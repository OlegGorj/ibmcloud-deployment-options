关于 单个和可扩展容器

IBM® Bluemix® 在所选区域中将 IBM Bluemix Container Service 基础架构作为一种应用程序开发功能来提供。通过容器，您可使用任何编程工具以任何语言构建应用程序。每个容器都是一个独立、安全的应用程序平台。容器可以在任意位置运行，例如运行 OS X 或 Windows 的开发工作站、运行 Ubuntu 的服务器、运行 Red Hat 的生产数据中心虚拟机，或者像 Bluemix 这样的云平台。

什么是容器？

容器是虚拟软件对象，其中包含应用程序运行所需的所有元素。容器是基于映像构建的，映像是用于创建容器的只读模板。每个映像都只包含应用程序及其依赖项，将作为独立的进程在主机操作系统上运行。因此，容器具备资源隔离和分配的优点，而与虚拟机等相比，却又更易于移植、更高效。容器可帮助您快速构建高质量的应用程序。

容器灵活

容器通过为开发和生产团队提供标准化环境，简化了系统管理。引擎的轻量级运行时支持快速向上扩展和向下扩展来响应需求的变化。容器有助于使管理不同操作系统平台和底层基础架构更加容易。容器可帮助您在任何基础架构上迅速、可靠地部署和运行任何应用程序。

容器小巧

在单个虚拟机所需的空间中，可以放入多个容器。容器的大小是可选择的，以便您只使用应用程序所需的空间量。

容器可移植

可以通过将其他映像用作基本映像，为其他容器构建映像。让其他人来做大部分的映像构建工作，您只需稍作调整即可使用。还可以快速将应用程序代码从过渡环境迁移到生产环境。迁移过程可通过工具（例如 Delivery Pipeline 或 UrbanCode Deploy）自动执行。

Docker 和 IBM Bluemix Container Service

IBM Bluemix Container Service 是一种利用 Docker 技术的 Bluemix 运行时。现有 Docker 用户和 Docker 新用户都可以使用 IBM Bluemix Container Service 的以下功能。

  • 将现有 Docker Hub 映像添加到 IBM 所托管的专用注册表中。
  • 验证映像是否有潜在漏洞。
  • 为 IBM Bluemix Container Service 配置命令行界面 (CLI)，以使用惯用的 Docker CLI 命令。
  • 无需任何其他配置，即可查看容器的日志记录和监视数据。

如果以前使用过 Docker，那么您可能会发现开始使用 IBM Bluemix Container Service 时，以下常见问题的答案非常有帮助。

如何在 IBM Bluemix Container Service 中使用 Docker Hub 的映像？
使用 IBM Bluemix Container Service CLI，可以复制 Docker Hub 的映像到专用 Bluemix 注册表中。复制映像后，即可以将其用于创建容器。
如何创建高可用性容器？
IBM Bluemix Container Service 提供了类似于 Docker 容器的单个容器。此外，IBM Bluemix Container Service 还提供了可扩展容器组，可用于使应用程序具有高可用性。容器组提供内置负载均衡器，可确保在容器实例之间路由工作负载并对其进行均衡。要增加组的可用性，可跨具有反亲缘关系的多个物理计算主机来散布容器组实例，并对运行状况欠佳的实例启用自动恢复。 此外，您还可以设置策略，以便根据 CPU 和内存的使用情况来自动扩展容器组实例。有关更多信息，请参阅将高可用性进程作为容器组运行。
如何将容器端口映射到主机端口？
IBM Bluemix Container Service 会区分专用容器网络与公用网络。在专用容器网络上，缺省情况下所有容器端口都会公开，无需额外配置。连接到相同专用容器网络的单个容器和容器组可以使用其分配的专用 IP 地址进行通信。无需进行端口映射。

要使应用程序可供公用网络使用，您可以公开公共端口，并将公共 IP 地址绑定到单个容器，或将公共路径绑定到容器组。公开公共端口允许您仅在公开的端口上发送和接收公共数据。其他所有公共端口都会关闭。

如何使容器在因特网上公共可用？
  • 对于单个容器，公开一个或多个端口，并向容器分配一个浮动 IP 地址。
  • 对于容器组，公开单个 HTTP 端口并分配路径。
是什么原因造成在本地 Docker 和 Bluemix 中构建容器所耗用的时间不同？
您可能会注意到使用本地 Docker 安装构建容器要比在 Bluemix 中构建容器更快。对于第一次部署，容器映像必须下载到主机上的注册表。映像的后续部署会更快。有关不同容器配置怎样影响部署速度的更多信息，请参阅容器类型。
如何在专用网络上创建容器，而不以公共方式将其公开？
创建容器后，该容器会连接到 IBM Bluemix Container Service 缺省容器专用网络，并且会向容器分配专用 IP 地址。在此专用网络上，缺省情况下所有容器端口都会公开。无需将主机端口映射到容器端口。要阻止从因特网访问容器，请不要在创建容器时绑定 IP 地址或公开端口。
如何将卷安装到容器？
根据设计，单个容器的生命周期很短。但是，可以使用卷在两次容器重新启动之间持久存储数据。在 Docker 中安装卷时，该卷会安装到本地文件系统。在 IBM Bluemix Container Service 中，限制了对计算主机的访问，因此无法将主机目录安装到容器。将改为使用组织范围的卷在两次容器重新启动之间持久存储数据。卷是在隔离的文件共享上进行托管的，这些文件共享用于安全存储应用程序数据，并管理对文件的访问权和许可权。要将卷安装到容器，您必须先创建卷。当您通过 CLI 创建容器时，使用 --volume 选项安装现有卷。从 Bluemix GUI 中，选择高级选项并分配现有卷。有关更多信息，请参阅使用命令行界面 (CLI) 添加文件到卷。
如何将 Docker Compose 与 IBM Bluemix Container Service 配合使用？
可以执行 Docker Compose 命令来通过 IBM Bluemix Container Service 创建并管理多容器部署。有关更多信息，请参阅教程：使用 Docker Compose 和 IBM Bluemix Container Service 创建多容器部署。
如何设置 Docker Compose 客户机，使其指向 IBM Containers 服务？
当您登录到 IBM Bluemix Container Service CLI 时，配置 CLI 运行本机 Docker 命令。有关更多信息，请参阅登录到 bx ic 插件。
如何登录到正在运行的容器？通过 SSH 登录吗？
您可以使用 bx ic exec 来登录到容器。
为什么我的容器以拆离模式运行？
缺省情况下，Bluemix 中的容器以拆离模式运行，以允许具有正在运行过程的容器在背景运行而无需退出。 以拆离模式运行的容器是云环境的标准模型。 通过运行 bx ic attach [--no-stdin] [--sig-proxy] CONTAINER，您仍可以连接到容器。

Bluemix Dedicated 和 Bluemix Local 中的 IBM Bluemix Container Service

Bluemix Dedicated 和 Bluemix Local 中提供了 IBM Bluemix Container Service。

Bluemix Local 中 IBM Bluemix Container Service 的硬件需求

基本硬件需求
缺省情况下，基本设置配置为 64GB 内存的容器计算能力。
  1. 您的环境必须满足 Bluemix 本地硬件需求。
  2. 要使用 IBM Bluemix Container Service，每一个设置都需要满足以下需求的控制平面。 此外，还需要两个实例才能进行监视和记录，它们满足相同的硬件需求。
    • VMware：3 个 ESX 服务器，每个具有 256GB 内存和 32 个物理核心（64 vCore 且启用超线程）。跨这 3 个服务器，需要 4TB 的集群共享存储器。
对增加额外硬件的需求
您可以通过添加达到 12 个扩展 VM，将 16GB 增量的容器计算能力增加到 256GB，而不增加硬件。 每个 VM 必须具有 20GB 内存、4 vCPU 和 200GB 磁盘空间。 但是，对于在集群中部署的每 10 个扩展 VM，需要 1 个额外的 VM 来处理网络操作。 此处理网络操作的 VM 需要 4vCPU、16GB 内存和 250GB 磁盘空间。
