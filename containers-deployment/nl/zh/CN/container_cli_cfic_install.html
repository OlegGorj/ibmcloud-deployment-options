设置 IBM Bluemix Container Service 插件 (bx ic) 以对单个和可扩展容器使用本机 Docker CLI

安装 IBM® Bluemix® Container Service 插件 (bx ic)，以运行本机 Docker CLI 命令来管理容器。
注意： 单个和可扩展容器 的免费试用版不可用。 了解有关集群管理和在 Bluemix Public 上迁移至 IBM Bluemix Container Service 中的 Kubernetes 的信息。

安装 IBM Bluemix Container Service 插件 (bx ic)

要运行命令，您的操作系统必须满足以下需求。此任务包含用于安装这些所需工具的信息。这些工具的较低版本无法与 IBM Bluemix Container Service 插件 (bx ic) 配合使用。

  • Docker V1.8 到 V1.12
  • Cloud Foundry CLI V6.14.0 - 6.22.0
    注： MacOS Sierra 用户必须安装 Cloud Foundry CLI V6.22.0 或更新版本。
  • Bluemix CLI
  • IBM Bluemix Container Service 插件

要安装 CLI，请执行以下操作：

  1. 安装 Docker V1.12。 对于 Windows 和 OS X 操作系统，Docker 会随 Docker Toolbox 一起安装，其中包含 Docker Machine、Docker Engine 和 Kitematic。对于 Linux 操作系统，安装机制根据使用的 Linux 分发版的类型不同而变化。请参阅以下链接以安装 Docker V1.12；或者如果已安装 Docker，请升级到该版本。
    注： 如果已在计算机上安装 Docker，但并不想升级当前版本，那么可以跳过此步骤，继续安装 Cloud Foundry CLI。
    • OS X

      https://github.com/docker/toolbox/releases/download/vV1.12/DockerToolbox-V1.12.pkg

    • Windows

      https://github.com/docker/toolbox/releases/download/vV1.12/DockerToolbox-V1.12.exe

    • Linux

      请参阅 Docker 安装文档以查找有关如何在使用的 Linux 分发版上安装 Docker VV1.12。

  2. 启动 Docker。
    • 对于 OS X 和 Windows，双击 Docker 快速启动终端图标。此时将启动 Docker 守护程序，并为您设置环境变量。
    • 对于 Linux，在命令行实用程序中，验证 Docker 守护程序是否在运行。此外，将 Docker 守护程序设置为自动重新启动。有关更多信息，请参阅适用于您的操作系统的 Docker 文档。
  3. 通过在终端运行 docker run hello-word，验证 Docker 安装已正确设置。 
    docker run hello-world
    Docker 会在本地查找 hello-world 映像，以从中运行容器，因此如果这是您首次运行该命令，那么将会收到以下消息。 
    Unable to find image 'hello-world:latest' locally
    映像是从公共映像存储库 Docker Hub 中拉取的，该命令完成后，输出将类似于以下内容： 
    $ docker run hello-world

Hello from Docker.
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.

To try something more ambitious, you can run an Ubuntu container with:
 $ docker run -it ubuntu bash

Share images, automate workflows, and more with a free Docker Hub account:
 https://hub.docker.com

For more examples and ideas, visit:
 https://docs.docker.com/userguide/
    如果命令未正确完成，请运行 docker version，以查看是否正确安装 Docker。
  4. 从 GitHub 存储库安装 Cloud Foundry CLI VCloud Foundry CLI V6.14.0 - 6.22.0。 必须将 Cloud Foundry CLI 安装在操作系统的缺省位置，否则 PATH 环境变量会与安装目录不匹配。
    注： MacOS Sierra 用户必须安装 Cloud Foundry CLI V6.22.0 或更新版本。
    要验证 Cloud Foundry CLI 是否已正确安装，或者要检查所安装的 Cloud Foundry CLI 是哪个版本，请运行以下命令：
    cf -v
  5. 安装 Bluemix CLI。
  6. 登录到 Bluemix CLI。 
    bx login -a https://api.ng.bluemix.net
  7. 安装 IBM Bluemix Container Service 插件。 
    bx plugin install IBM-Containers -r Bluemix
    要验证是否已正确安装该插件，请运行以下命令：
    bx plugin list

    该插件会在结果中显示为 IBM-Containers。

您可随时登录并开始管理容器。

登录到 IBM Bluemix Container Service CLI 插件 (bx ic)

安装 CLI 后，需要登录才能使用。

开始之前，请完成以下任务。
  • 安装 IBM Bluemix Container Service (bx ic) 插件。
  • 决定在登录到 Cloud Foundry 时要使用的 Bluemix 组织和空间。 有关组织和空间的更多信息，请参阅组织和空间。
  • 有关可在 Cloud Foundry login 命令中指定哪些选项的更多信息，请参阅 Cloud Foundry 文档中的 Login 命令。
  1. 打开命令行实用程序。
    • 对于 OS X 和 Windows，双击 Docker 快速启动终端图标。此时将启动 Docker 守护程序，并为您设置环境变量。
    • 对于 Linux，打开任何命令行实用程序。
  2. 通过 Cloud Foundry CLI 登录到 Bluemix。 要指定特定区域，请使用 -a 选项来指定该区域的 API 端点。第一次使用 CLI 时，如果未指定该选项，那么系统会提示您输入 API 端点。指定 API 端点后，在后续登录时就无需输入 API 端点。 
    bx login [-a api.DomainName] [-sso]
    要登录到特定区域，请在登录到 Cloud Foundry CLI 时包括该区域的 API 端点。
    • 美国南部
      bx login -a api.ng.bluemix.net
    • 英国
      bx login -a api.eu-gb.bluemix.net
    当您使用联合标识登录时，需要单点登录参数 --sso。如果使用此选项，请打开 https://login.ng.bluemix.net/UAALoginServerWAR/passcode，以获取一次性密码。如果您未包括 --sso 选项，请完成以下子步骤。
    1. 对于电子邮件，请输入您用于 Bluemix 的 IBM 标识。
    2. 对于密码，请输入 IBM 标识的密码。 此时将检索您的 Bluemix 组织和空间。
    3. 输入代表您某个 Bluemix 组织的数字。
    4. 输入代表其中一个现有 Bluemix 空间的数字。
  3. 设置组织的名称空间。 名称空间是用于识别专用 Bluemix 映像注册表的唯一名称。创建容器时，必须通过包含具有映像名称的名称空间来指定映像的位置。
    注： 创建名称空间时，在创建后即不能进行更改，只能删除。 如果您 Bluemix 组织中的用户已经部署容器并设置名称空间，那么会显示消息，其中显示您组织的名称空间。 在此情况下，您可以继续执行下一步。
    bx ic namespace-set <my_namespace>
    提示： 请遵循名称空间的以下规则：
    • 名称只能包含小写字母、数字或下划线 (_)。
    • 名称可以为 4 到 30 个字符。如果计划从命令行管理容器，那么您可能倾向于使用能够快速输入的简短名称空间。
    • 其名称在 Bluemix 中必须是唯一的。
  4. 初始化 IBM Bluemix Container Service。 
    bx ic init
  5. 决定您是要运行 bx ic 还是本机 Docker 命令。 您可以在每次登录到插件时选择以下某个选项，也可以使用 shellinit 命令以始终选择“选项 2”。
    • 选项 1（缺省）

      IBM Bluemix Container Service 包装程序 Docker 的简单 shell。本地 Docker 守护程序不受 IBM Bluemix Container Service 的影响。bx ic 命令用于在 Bluemix 空间中管理容器，而 docker 命令用于在本地管理容器。但是，本地映像开发始终需要 docker 命令。

      例如，使用“选项 1”，bx ic run 命令会在 Bluemix 空间创建容器，而 docker run 会在本地创建容器。如果您对 Docker 和 IBM Bluemix Container Service 来说是新手，或者您想要在本地测试容器，那么您可能更愿意使用此选项。

    • 选项 2

      将当前 CLI 会话的本地 Docker 守护程序重定向到 IBM Bluemix Container Service，以便在 docker 命令可用于在 Bluemix 空间中管理容器时，替换 bx ic 命令。

      例如，使用“选项 2”，bx ic run 或 docker run 命令可用于在您的空间中创建容器。如果您是经验丰富的 Docker 用户，熟悉 docker 命令，或者您编写自动化脚本来使用 IBM Bluemix Container Service，那么您可能更愿意使用此选项。

      某些例外情况仍需要使用 bx ic 命令来执行特定于 IBM Bluemix Container Service 的任务。例如，要设置名称空间，您必须使用命令 bx ic namespace-set。docker namespace-set 不是有效的命令。要获取此配置支持的 docker 命令列表，请参阅IBM Bluemix Container Service 插件 (bx ic) 支持的 Docker 命令。

      要配置此选项，请复制并粘贴登录期间终端中提供的环境变量。
      • DOCKER_HOST 用于将 Docker 主机设置为 IBM Bluemix Container Service。
      • DOCKER_CERT_PATH 用于设置 IBM Bluemix Container Service Cloud Foundry 插件的认证密钥位置。
      • DOCKER_TLS_VERIFY 使用 TLS 并验证远程服务器。
      提示： 完成 IBM Bluemix Container Service 文档中的任务后，就可以在标记有星号 (*) 的所有步骤中运行 docker 命令。

升级 IBM Bluemix Container Service 插件 (bx ic)

您可能想要定期更新插件以使用新功能。

表 1. IBM Bluemix Container Service 插件的当前版本
发布日期 版本 更新
2016 年 11 月 16 日 0.8.964
  • 添加了通过 bx ic group-create 命令，使用 --http_monitor_enabled、--http_monitor_path 和 --http_monitor_rs_list 选项管理容器组的运行状况检查的支持
  • 添加了对 bx ic group-create --session_affinity 选项的支持
  • 缺陷修订

要更新插件，请执行以下步骤。

  1. 通过首次安装插件时配置的插件存储库来安装更新。 
    bx plugin update IBM-Containers -r Bluemix
  2. 通过运行以下命令并检查已安装的插件的列表来验证插件安装。 
    bx plugin list

    该插件会在结果中显示为 IBM-Containers。

卸载 IBM Bluemix Container Service 插件 (bx ic)

如果不再需要 IBM Bluemix Container Service 插件，可以使用 bx 命令将其卸载。

注： 要卸载 IBM Bluemix Container Service Cloud Foundry 插件，请运行以下命令。
cf uninstall-plugin IBM-Containers
  1. 要为 Bluemix CLI 卸载 IBM Bluemix Container Service 插件，请运行以下命令。 
    bx plugin uninstall IBM-Containers
  2. 通过运行以下命令并检查已安装的插件的列表，验证该插件是否已卸载。 
    bx plugin list

    IBM Bluemix Container Service 插件未显示在结果中。

安装 Docker Compose 及其依赖项

要同时启动和停止一些单个容器，可以安装 Docker Compose CLI。

要通过 IBM Bluemix Container Service 来运行 Docker Compose 命令，您必须安装以下软件。
  • Docker V1.10.0
  • Cloud Foundry CLI V6.14.0 - 6.22.0
  • Bluemix CLI
  • IBM Bluemix Container Service 插件
  • Docker Compose V1.6.0 或更高版本
此任务包含用于安装这些所需工具的信息。这些工具的较低版本无法与 IBM Bluemix Container Service 插件 (bx ic) 配合使用。
  1. 安装 Docker、Cloud Foundry CLI、Bluemix CLI 和 IBM Bluemix Container Service 插件。
    注： 如果使用的是 Mac 或 Windows 操作系统，Docker Compose 将作为 Docker Toolbox 安装的一部分，随 Docker Engine、Docker Machine 和 Kitematic 一起安装。您可跳过下一步，继续验证 Docker Compose 版本。
  2. 如果使用的是 Linux，请安装最新版本的 Docker Compose。
    1. 安装 pip。 pip 是软件包管理器，其中包含用于安装、升级和除去计算机上用 Python 编写的软件包的所有工具。您可使用以下命令来安装 pip，也可直接从 Python Package Index 将软件包下载到计算机。
      注： 如果已在计算机上安装 pip，那么可以继续安装最新版本的 Docker Compose。
      easy_install pip
    2. 安装 Docker Compose 
      pip install docker-compose==1.6.2
  3. 验证计算机上安装的 Docker Compose 版本。
    1. 登录到 IBM Bluemix Container Service CLI。 登录到该插件来运行本机 Docker 命令时，请设置选项 2 的环境变量。
    2. 验证 Docker Compose 包的版本。 
      docker-compose version
      如果显示 docker-compose version 1.6.0 或更高版本，那么可以开始创建自己的多容器应用程序。否则，请升级 Docker Compose 版本。
      注： 如果执行 Docker Compose 命令时遇到问题，请复查没有名为 queue 的模块和不安全 HTTPS 连接警告主题，以检查是否需要更改系统设置或安装其他软件包。

升级 Docker Compose

您可能需要升级 Docker Compose 包的版本，才能使用最新功能。

  1. 升级 Docker Compose。 
    pip install docker-compose --upgrade
  2. 验证 Docker Compose 包的版本。 
    docker-compose version

卸载 Docker Compose

如果不再需要 Docker Compose 软件包，可以使用以下命令将其卸载。

  1. 卸载 Docker Compose。 
    pip uninstall docker-compose
  2. 验证 Docker Compose 软件包是否已卸载。 如果 Docker Compose 已成功卸载，那么运行以下命令后，CLI 输出中不会列出 Docker Compose。
    pip freeze
