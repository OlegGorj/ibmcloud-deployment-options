bx ic attach [--no-stdin] [--sig-proxy] CONTAINER

用途
控制正在运行的容器或查看其输出。使用 CTRL+C 以退出并停止容器。有关更多信息，请参阅 Docker 帮助中的 attach 命令。
参数
--no-stdin
（可选）不包含标准输入。
--sig-proxy
（可选）缺省值为 true。
CONTAINER
（必需）容器名称或标识。
示例
以下示例是用于连接到容器 my_container 的请求。
bx ic attach my_container

bx ic build -t TAG [--no-cache] [--pull] [-q] DOCKERFILE_LOCATION

用途
在本地或在专用 Bluemix 映像注册表中构建 Docker 映像。有关更多信息，请参阅 Docker 帮助中的 build 命令。
参数
DOCKERFILE_LOCATION
（必需）本地主机上 Dockerfile 和上下文文件的路径。
--no-cache
（可选）首次构建映像时，会对整个组织的映像层进行高速缓存。构建映像的后续版本时，这些映像的高速缓存版本可用于加快构建过程。构建映像的新版本时，通过包含 --no-cache 选项，还可确保映像的每层都重新构建。
--pull
（可选）缺省情况下，仅当基本映像在组织的映像高速缓存中不存在时，才会从注册表中拉取基本映像。通过包含此选项，可以确保使用最新版本的基本映像来构建新映像。
-q, --quiet
（可选）禁止显示容器生成的详细输出。缺省值为 false。
-t TAG, --tag TAG
（必需）要应用到所创建映像的专用映像注册表的路径。
示例
以下示例是用于构建名为 myimage 的映像的请求。要在该构建中使用的 Dockerfile 和其他工件位于运行该命令的目录中。由于映像名称带有注册表和名称空间，因此会在组织的专用 Bluemix 映像注册表中构建映像。
提示： 运行 bx ic namespace-get 来检索您的名称空间，并将 <my_namespace> 替换为您的名称空间信息。
bx ic build -t registry.DomainName/<my_namespace>/myimage .

bx ic cp SRC_PATH CONTAINER:DEST_PATH | bx ic cp CONTAINER:SRC_PATH DEST_PATH

用途
将文件或文件夹从运行中或已停止的单个容器复制到本地计算机或反之。 有关更多信息，请参阅 Docker 帮助中的 cp 命令。
注： 仅支持此命令与单个容器一起使用。 当您想要从容器组复制文件或将文件复制到容器组，您必须从组中的每一个容器实例中复制文件或将文件复制到组中的每一个容器实例。
参数
DEST_PATH
（必需）要从源路径复制信息的本地计算机或容器上的文件或文件夹的路径。该文件或文件夹必须位于本地计算机或容器中，您才能开始复制。 如果您要复制到本地计算机上，那么您可以使用绝对或相对路径。 如果要复制到单个容器，那么您必须包括容器的名称、定界符 “:” 和目标文件或文件夹的路径。 路径必须相对于容器的根目录。
SRC_PATH
（必需）要复制的文件或文件夹的路径。如果文件或文件夹位于本地计算机上，那么您可以使用绝对或相对路径。 如果文件或文件夹位于单个容器中，那么您必须包括容器的名称、定界符 “:” 和要复制的文件或文件夹的路径。 路径必须相对于容器的根目录。
注： 您无法复制存储在容器 /dev、/etc、/proc 和 /sys 文件夹下的特定系统文件。
示例
此示例命令将名为 config.txt 的文件从本地计算机的当前目录复制到名为 my_container 的单个容器的 config 目录中。
bx ic cp ./config.txt my_container:/config

bx ic cpi SRC DST

用途
访问 Docker Hub 映像或本地注册表中的映像，然后将其复制到专用 Bluemix 映像注册表。
注： 此命令是同时使用 docker pull 和 docker push 将映像添加到 Bluemix 映像注册表的替代方法。如果您更愿意使用 docker pull 和 docker push，请参阅通过命令行将本地映像推送到专用 Bluemix 注册表。
参数
DST
（必需）专用 Bluemix 映像注册表 URL，其中包含名称空间和目标映像名称。映像的标记是可选的。
SRC
（必需）源存储库和映像名称。
示例
bx ic cpi source_repository/source_image_name private_registry_URL/destination_image_name:tag
提示： 运行 bx ic namespace-get 来检索您的名称空间，并将 <my_namespace> 替换为您的名称空间信息。
bx ic cpi training/sinatra registry.DomainName/<my_namespace>/mysinatra:v1

bx ic exec [-d] [-it ] [-u USER] CONTAINER CMD

用途
在容器内执行命令。有关更多信息，请参阅 Docker 帮助中的 exec 命令。
参数
CMD
（必需）要在指定的一个或多个容器内执行的命令。
CONTAINER
（必需）容器名称或标识。
-d , --detach
（可选）在后台运行指定的命令。
-it
（可选）交互方式。使标准输入保持显示。输入 exit 以退出。
-u USER, --user USER
（可选）用户名
示例
此示例命令将在容器 my_container 中运行交互式 bash 终端。
bx ic exec -it my_container bash
此示例命令将在容器 my_container 中运行 date 命令。
bx ic exec my_container date

bx ic group-create [--anti] [--auto] [-d DOMAIN] [--desired DESIRED] [-e ENV] [--env-file ENVFILE] [--http_monitor_enabled] [--http_monitor_path] [--http_monitor_rc_list] [--ip IP_ADDRESS] [-m MEMORY] [--max MAX] [--min MIN] [-n HOST] --name NAME [-p PORT] [--session-affinity] [--volume VOLUME:/DIRECTORY_PATH] IMAGE [CMD]

用途
创建扩展组。缺省情况下，Bluemix 中的容器以拆离模式运行。
参数
--anti
（可选）使用反亲缘关系，使您的容器组具有更高的可用性。--anti 选项可强制组中的每一个容器实例置于不同的物理计算节点上，这可降低组中所有容器因硬件故障而发生崩溃的几率。 您可能无法对较大的组大小使用此选项，因为每一个 Bluemix 区域和组织可用于部署的计算节点集有限制。 如果部署不成功，请减少组中容器实例的数目，或者除去 --anti 选项。
--auto
（可选）针对容器组启用了自动恢复并且该组已运行了 10 分钟时，该组的负载均衡器会开始通过 HTTP 请求定期检查组中每个容器实例的运行状况。如果容器实例在 100 秒内未响应，即会将该实例标记为“不活动”。自动恢复会从组中除去不活动的容器实例并重新创建这些实例。自动恢复会尝试恢复组中的容器实例 3 次。在第三次尝试后，自动恢复在 60 分钟内不会再恢复组中的任何容器实例。60 分钟后，自动恢复进程将再次启动。有关运行状况检查的更多信息，请参阅 --http_monitor_enabled 选项。
CMD
（可选）该命令及相应自变量将传递给容器组来执行。此命令必须是长时间运行命令。请不要使用短时间运行命令（例如 /bin/date），因为此类命令可能会导致容器崩溃。
长时间运行命令的示例（使用 IBM Bluemix Container Service 提供的 ibmnode 映像）：
bx ic group-create --name my_container_group registry.DomainName/ibmnode ping localhost
bx ic group-create --name my_container_group registry.DomainName/ibmnode -- tail -f /dev/null
注： CMD 及其自变量必须位于 bx ic group-create 命令行的末尾。
-d DOMAIN, --domain DOMAIN
（可选）缺省系统域为 AppDomainName，此域已提供 SSL 证书，所以可以使用 HTTPS 来访问容器组，而无需任何额外配置。要使用定制域，必须在公共 DNS 服务器上注册定制域，在 Bluemix 中配置定制域，然后将定制域映射到公共 DNS 服务器上的 Bluemix 系统域。 定制域映射到 Bluemix 系统域后，对定制域的请求会路由到 Bluemix 中的应用程序。创建定制域时，不要在域名中包含下划线 (_)。有关定制域的更多信息，请参阅创建和使用定制域。为了保护定制域安全，请上传 SSL 证书，以便可以使用 HTTPS 访问容器组。

主机和域组合在一起构成了完整的公共路径 URL（例如 http://mycontainerhost.AppDomainName），并且该 URL 在 Bluemix 中必须唯一。使用 bx ic group-inspect 命令来查看容器组的详细信息时，主机和域会作为路径一起列出。

--desired DESIRED
（可选）组中的实例数。缺省值为 2。应用程序所需的容器组实例数取决于期望的工作负载和应用程序可以处理请求的时间。 例如，多任务或网络连接会影响应用程序所需的实例数。 要确定所需实例的数目，可在运行应用程序的 Bluemix 中部署单个容器，并在此容器上执行负载测试。例如，如果应用程序每秒可处理 100 个请求，但是您期望每秒平均工作负载为 300 个请求，那么容器组的所需实例数目为 3。 要使容器组具有高可用性且对故障的处理更富弹性，请考虑除最低要求的实例数外，额外包含更多实例，以用于处理预期的工作负载。额外的实例可在其他实例崩溃时处理工作负载。要确保自动恢复发生故障的实例，而不影响工作负载，请包含一个额外实例。要避免两个实例同时发生故障，请包含两个额外实例。此设置是 N+2 模式，其中 N 是处理请求的实例数，+2 是额外两个实例。
要点： 容器组应该至少包含 2 个容器实例。 如果组中仅包含 1 个实例，且该实例变得不可用，那么用户可能会遭遇应用程序停机时间。
-e ENV, --env ENV
（可选）设置环境变量，其中 ENV 是 key=value 对。如果包含引号，并用引号将环境变量名称和值括起，那么将单独列出多个键。示例：-e “key1=value1” -e “key2=value2a,value2b” -e “key3=value3”

--http_monitor_enabled
（可选）缺省值为 true。针对容器组启用了自动恢复并且该组已运行了 10 分钟时，该组的负载均衡器会开始通过 HTTP 请求定期检查组中每个容器实例的运行状况。如果容器实例在 100 秒内未响应，即会将该实例标记为“不活动”。自动恢复会从组中除去不活动的容器实例并重新创建这些实例。自动恢复会尝试恢复组中的容器实例 3 次。在第三次尝试后，自动恢复在 60 分钟内不会再恢复组中的任何容器实例。60 分钟后，自动恢复进程将再次启动。一些应用程序可能不需要进行 HTTP 运行状况检查，如使用 TCP 而非 HTTP 进行通信的应用程序。要禁用 HTTP 监视器，请将该选项设置为 false。示例：--http_monitor_enabled=false
--http_monitor_path
（可选）缺省值为 /。当启用 --http_monitor_enabled 时，您可以使用此选项来设置运行状况检查的定制路径，如 --http_monitor_path=“/health/path”。
--http_monitor_rc_list
（可选）当启用 --http_monitor_enabled 时，您可以指定运行状况检查的响应代码列表。 缺省值包括 200、201、202、204、300、301、302、401、403 和 404 代码。如果出现不在有效响应列表中的任何响应，那么负载均衡器会将容器实例状态更改为 INACTIVE。处于不活动状态的容器不会接收任何负载，而容器组的状态会更改为 HAS_INACTIVE_INSTANCES。如果没有容器处于 ACTIVE 状态，那么容器组的状态会更改为 NO_ACTIVE_INSTANCES。要了解状态的更多信息，请参阅IBM Bluemix Container Service 中的容器状态。
IMAGE
（必需）要包含在容器组中的每个容器实例内的映像。可以在映像之后列出命令，但不要在映像之后放置任何选项；请在指定映像之前包含所有选项。

如果要在组织的专用 Bluemix 映像注册表中使用映像，请使用以下格式指定映像：registry.DomainName/NAMESPACE/IMAGE

如果您要使用 IBM Bluemix Container Service 提供的映像，请不要包含组织的名称空间。请使用以下格式指定映像：registry.DomainName/IMAGE

--ip IP_ADDRESS
（可选）如果您有可用的 IP 地址，请将公共 IP 地址绑定到容器组。此外，还必须使用 -p 标志来公开端口。容器组可以具有公共路径或公共 IP 地址，但不能同时具有公共路径和公共 IP 地址。要取消绑定 IP 地址，并将该 IP 还给空间的配额，请除去容器组，然后重新创建不使用该 IP 地址的容器组。要在创建容器组之前请求 IP 地址，请运行 bx ic ip-request。
注： 绑定公共 IP 地址不会包含 SSL 证书，此证书用于加密发送到容器组和从容器组发送的数据。如果应用程序需要 SSL 加密，那么可以实施您自己的 SSL 解决方案，或者将公共路径映射到容器组，而不绑定公共 IP 地址。公共路径已经包含 SSL 证书，所以可以使用 HTTPS 访问容器组，而无需任何其他配置。
-m MEMORY, --memory MEMORY
（可选）输入容器的内存限制 (MB)。内存限制是容器大小的一部分，用于定义在运行时容器在计算主机上所获取的内存和磁盘空间的最大量。 分配容器大小后，此值无法更改。IBM Bluemix Container Service 中可用的大小为：
  • Pico（64 MB 内存，4 GB 磁盘空间）
  • Nano（128 MB 内存，8 GB 磁盘空间）
  • Micro（256 MB 内存，16 GB 磁盘空间）
  • Tiny（512 MB 内存，32 GB 磁盘空间）
  • Small（1024 MB 内存，64 GB 磁盘空间）
  • Medium（2048 MB 内存，128 GB 磁盘空间）
  • Large（4096 MB 内存，256 GB 磁盘空间）
  • X-Large（8192 MB 内存，512 GB 磁盘空间）
  • 2X-Large（16384 MB 内存，1 TB 磁盘空间）
如果您未设置容器的大小，那么会使用 256 MB 创建每一个容器实例。
要点： 在没有单位标签的情况下，输入容器内存 (MB)。例如，如果您想要创建 Pico 容器，请输入 -m 64。
--max MAX
（可选）最大实例数。缺省值为 2。
--min MIN
（可选）最小实例数。缺省值为 1。
-n HOST, --hostname HOST
（可选）主机名，例如 mycontainerhost。不要在主机名中包含下划线 (_)。主机和域组合在一起构成了完整的公共路径 URL，例如 http://mycontainerhost.AppDomainName。使用 bx ic group-inspect 命令来查看容器组的详细信息时，主机和域会作为路径一起列出。
--name NAME
（必需）为组分配名称。不推荐使用 -n。
提示： 容器名称必须以字母开头，并且可以包含大写字母、小写字母、数字、句点 (.)、下划线 (_) 或连字符 (-)。
-p PORT, --publish PORT

（必需）显示用于 HTTP 和专用网络流量的端口。如果您未显示端口，虽然容器组会运行，但是无法通过公用或专用网络上的组负载均衡器来访问容器组。

对于容器组，不能包含多个端口。如果列出多个端口，那么仅会公开最后一个列出的端口。该命令会忽略其他端口。

绑定路径时，组中的容器必须在组的公开端口上侦听 HTTP 流量。非 HTTP 端口不能向公用网络公开。HTTPS 流量到达公开的端口时，(Go)Router 会完成 HTTPS 终止。随后，专用虚拟网络上 (Go)Router 和容器之间的连接将使用 HTTP 协议。绑定 IP 地址时，组中的容器可以侦听 HTTP 或 HTTPS 流量，但您仍然只能公开一个端口。
如果在 Dockerfile 中为要使用的映像指定了端口，请包含该端口。
提示：

  • Micro（256 MB 内存，16 GB 磁盘空间）
  • Tiny（512 MB 内存，32 GB 磁盘空间）

  • Medium（2048 MB 内存，128 GB 磁盘空间）
  • Large（4096 MB 内存，256 GB 磁盘空间）
  • X-Large（8192 MB 内存，512 GB 磁盘空间）
  • 2X-Large（16384 MB 内存，1 TB 磁盘空间）

-p PORT, --publish PORT

（可选）如果要使应用程序可通过因特网进行访问，您必须公开公共端口。公开公共端口时，将为容器创建公用网络安全组，该组只允许您在公开的端口上发送和接收公共数据。其他所有公共端口都将关闭，且无法用于从因特网访问应用程序。可以使用多个 -p 选项包含多个端口。端口无法映射或转发。如果未公开端口，那么只能从专用容器网络访问容器。可以使用分配的专用 IP 地址在专用网络上与容器通信。

如果在 Dockerfile 中为要使用的映像指定了端口，请包含该端口。
提示：
  • 对于 IBM 认证的 Liberty Server 映像或此映像的修改版本，请输入端口 9080。
  • 对于 IBM 认证的 Node.js 映像或此映像的修改版本，请输入任意端口，例如端口 8000。

您可以选择 UDP 或 TCP，以指示要使用的 IP 协议。如果未指定协议，那么端口会自动公开用于 TCP 流量。

示例
  • -p 9080
  • -p 9080/udp
  • -p 9080/udp -o 8000/tcp

如果空间中有要绑定到容器的现有 IP 地址，那么可以指定该 IP 地址以及端口信息。请包含 IP 地址、端口或 IP 地址和容器端口号，例如 <ip-address>:<port_on_IP>:<container-port>。IP 地址上的端口必须与 Dockerfile 中公开的容器端口相匹配。

示例
-p 192.0.2.23:9080:9080
注： 绑定公共 IP 地址不会包含 SSL 证书，此证书用于加密发送到单个容器和从单个容器发送的数据。如果应用程序需要 SSL 加密，那么可以实施您自己的 SSL 解决方案，或者使用容器组，而不使用单个容器。容器组已绑定到已经包含 SSL 证书的公共路径，所以可以使用 HTTPS 来访问容器组，而无需任何额外配置。有关更多信息，请参阅使用 Bluemix GUI 创建容器组。

cf ic group create [--anti] [--auto] [-d DOMAIN] [--desired DESIRED] [-e ENV] [--env-file ENVFILE] [--http_monitor_enabled] [--http_monitor_path] [--http_monitor_rc_list] [--ip IP_ADDRESS] [-m MEMORY] [--max MAX] [--min MIN] [-n HOST] --name NAME [-p PORT] [--session_affinity] [--volume VOLUME:/DIRECTORY_PATH] IMAGE [CMD]

用途
创建扩展组。缺省情况下，Bluemix 中的容器以拆离模式运行。
参数
--anti
（可选）使用反亲缘关系，使您的容器组具有更高的可用性。--anti 选项可强制组中的每一个容器实例置于不同的物理计算节点上，这可降低组中所有容器因硬件故障而发生崩溃的几率。 您可能无法对较大的组大小使用此选项，因为每一个 Bluemix 区域和组织可用于部署的计算节点集有限制。 如果部署不成功，请减少组中容器实例的数目，或者除去 --anti 选项。
--auto
（可选）针对容器组启用了自动恢复并且该组已运行了 10 分钟时，该组的负载均衡器会开始通过 HTTP 请求定期检查组中每个容器实例的运行状况。如果容器实例在 100 秒内未响应，即会将该实例标记为“不活动”。自动恢复会从组中除去不活动的容器实例并重新创建这些实例。自动恢复会尝试恢复组中的容器实例 3 次。在第三次尝试后，自动恢复在 60 分钟内不会再恢复组中的任何容器实例。60 分钟后，自动恢复进程将再次启动。有关运行状况检查的更多信息，请参阅 --http_monitor_enabled 选项。
CMD
（可选）该命令及相应自变量将传递给容器组来执行。此命令必须是长时间运行命令。请不要使用短时间运行命令（例如 /bin/date），因为此类命令可能会导致容器崩溃。
长时间运行命令的示例（使用 IBM Bluemix Container Service 提供的 ibmnode 映像）：
cf ic group create --name my_container_group registry.DomainName/ibmnode ping localhost
cf ic group create --name my_container_group registry.DomainName/ibmnode -- tail -f /dev/null
注： CMD 及其自变量必须位于 cf ic group create 命令行的末尾。

（可选）缺省系统域为 AppDomainName，此域已提供 SSL 证书，所以可以使用 HTTPS 来访问容器组，而无需任何额外配置。要使用定制域，必须在公共 DNS 服务器上注册定制域，在 Bluemix 中配置定制域，然后将定制域映射到公共 DNS 服务器上的 Bluemix 系统域。 定制域映射到 Bluemix 系统域后，对定制域的请求会路由到 Bluemix 中的应用程序。创建定制域时，不要在域名中包含下划线 (_)。有关定制域的更多信息，请参阅创建和使用定制域。为了保护定制域安全，请上传 SSL 证书，以便可以使用 HTTPS 访问容器组。

主机和域组合在一起构成了完整的公共路径 URL（例如 http://mycontainerhost.AppDomainName），并且该 URL 在 Bluemix 中必须唯一。使用 cf ic group inspect 命令来查看容器组的详细信息时，主机和域会作为路径一起列出。

--desired DESIRED
（可选）组中的实例数。缺省值为 2。应用程序所需的容器组实例数取决于期望的工作负载和应用程序可以处理请求的时间。 例如，多任务或网络连接会影响应用程序所需的实例数。 要确定所需实例的数目，可在运行应用程序的 Bluemix 中部署单个容器，并在此容器上执行负载测试。例如，如果应用程序每秒可处理 100 个请求，但是您期望每秒平均工作负载为 300 个请求，那么容器组的所需实例数目为 3。 要使容器组具有高可用性且对故障的处理更富弹性，请考虑除最低要求的实例数外，额外包含更多实例，以用于处理预期的工作负载。额外的实例可在其他实例崩溃时处理工作负载。要确保自动恢复发生故障的实例，而不影响工作负载，请包含一个额外实例。要避免两个实例同时发生故障，请包含两个额外实例。此设置是 N+2 模式，其中 N 是处理请求的实例数，+2 是额外两个实例。
要点： 容器组应该至少包含 2 个容器实例。 如果组中仅包含 1 个实例，且该实例变得不可用，那么用户可能会遭遇应用程序停机时间。

（可选）设置环境变量，其中 ENV 是 key=value 对。如果包含引号，并用引号将环境变量名称和值括起，那么将单独列出多个键。示例：-e “key1=value1” -e “key2=value2a,value2b” -e “key3=value3”

  • -p 9080/udp
  • -p 9080/udp -o 8000/tcp
