bx ic attach [--no-stdin] [--sig-proxy] CONTAINER

用途
控制執行中容器，或檢視其輸出。使用 CTRL+C 以結束並停止容器。如需相關資訊，請參閱 Docker 說明中的 attach 指令。
參數
--no-stdin
（選用）不包括標準輸入。
--sig-proxy
（選用）預設值為 true。
CONTAINER
（必要）容器名稱或 ID。
範例
下列範例要求會連接至容器 my_container。
bx ic attach my_container

bx ic build -t TAG [--no-cache] [--pull] [-q] DOCKERFILE_LOCATION

用途
在本端或在專用 Bluemix 映像檔登錄中建置 Docker 映像檔。如需相關資訊，請參閱 Docker 說明中的 build 指令。
參數
DOCKERFILE_LOCATION
（必要）本端主機上 Dockerfile 及環境定義的路徑。
--no-cache
（選用）第一次建置映像檔時，會快取整個組織的映像檔層。當您建置後續版本的映像檔時，會使用映像檔的這些快取版本來加快建置處理程序。建置新版本的映像檔時包含 --no-cache 選項，即可確保一併重建映像檔的每一層。
--pull
（選用）依預設，唯有在您組織的映像檔快取中還沒有基礎映像檔時，才會從登錄中取回基礎映像檔。包含此選項，即可確保使用基礎映像檔的最新版本來建置新的映像檔。
-q、--quiet
（選用）抑制容器所產生的詳細輸出。預設值是 false。
-t TAG、--tag TAG
（必要）要套用至所建立映像檔之專用映像檔登錄的路徑。
範例
下列範例要求會建置名為 myimage 的映像檔。Dockerfile 及其他要在建置中使用的構件，位於執行指令所在處的同一目錄中。因為登錄及名稱空間隨附於映像檔名稱，所以映像檔會建置在您組織的專用 Bluemix 映像檔登錄中。
提示： 執行 bx ic namespace-get，以擷取名稱空間，並將 <my_namespace> 取代為您的名稱空間資訊。
bx ic build -t registry.DomainName/<my_namespace>/myimage .

bx ic cp SRC_PATH CONTAINER:DEST_PATH | bx ic cp CONTAINER:SRC_PATH DEST_PATH

用途
將檔案或資料夾從執行中或已停止的單一容器複製到本端機器，反之亦然。如需相關資訊，請參閱 Docker 說明中的 cp 指令。
註： 僅支援此指令與單一容器搭配使用。當您想要從容器群組複製檔案或將檔案複製到容器群組中時，必須從該群組的每一個容器實例中複製檔案，或將檔案複製到每一個容器實例中。
參數
DEST_PATH
（必要）本端機器或容器上您要在其中複製來源路徑之資訊的檔案或資料夾的路徑。檔案或資料夾必須存在於本端機器或容器上，然後您才能開始複製。如果您要複製到本端機器，可以使用絕對或相對路徑。如果您要複製到單一容器，則必須包含容器的名稱、定界字元 ":"，以及目的地檔案或資料夾的路徑。此路徑必須相對於容器的根目錄。
SRC_PATH
（必要）您要複製之檔案或資料夾的路徑。如果檔案或資料夾位於本端機器上，您可以使用絕對或相對路徑。如果檔案或資料夾位於單一容器中，您必須包含容器的名稱、定界字元 ":"，以及您要複製之檔案或資料夾的路徑。路徑必須相對於容器的根目錄。
註： 您無法複製容器的 /dev、/etc、/proc 及 /sys 資料夾下儲存的特定系統檔案。
範例
此範例指令會將名為 config.txt 的檔案，從本端機器的現行目錄複製到名為 my_container 之單一容器的 config 目錄。
bx ic cp ./config.txt my_container:/config

bx ic cpi SRC DST

用途
存取 Docker Hub 映像檔或本端登錄中的映像檔，並將它複製到您的專用 Bluemix 映像檔登錄。
註： 此指令是一起使用 docker pull 及 docker push 來將映像檔新增至 Bluemix 映像檔登錄的替代方案。如果您偏好使用 docker pull 及 docker push，請參閱從指令行將本端映像檔推送至專用 Bluemix 登錄。
參數
DST
（必要）專用 Bluemix 映像檔登錄 URL（包含名稱空間）及目的地映像檔名稱。映像檔的標籤是選用項目。
SRC
（必要）來源儲存庫及映像檔名稱。
範例
bx ic cpi source_repository/source_image_name private_registry_URL/destination_image_name:tag
提示： 執行 bx ic namespace-get，以擷取名稱空間，並將 <my_namespace> 取代為您的名稱空間資訊。
bx ic cpi training/sinatra registry.DomainName/<my_namespace>/mysinatra:v1

bx ic exec [-d] [-it ] [-u USER] CONTAINER CMD

用途
在容器內執行指令。如需相關資訊，請參閱 Docker 說明中的 exec 指令。
參數
CMD
（必要）要在所指定容器內執行的指令。
CONTAINER
（必要）容器名稱或 ID。
-d、--detach
（選用）在背景中執行指定的指令。
-it
（選用）互動模式。持續顯示標準輸入。鍵入 exit 以結束。
-u USER、--user USER
（選用）使用者名稱。
範例
此範例指令會在容器 my_container 中執行互動式 Bash 終端機。
bx ic exec -it my_container bash
此範例指令會在容器 my_container 中執行 date 指令。
bx ic exec my_container date

bx ic group-create [--anti] [--auto] [-d DOMAIN] [--desired DESIRED] [-e ENV] [--env-file ENVFILE] [--http_monitor_enabled] [--http_monitor_path] [--http_monitor_rc_list] [--ip IP_ADDRESS] [-m MEMORY] [--max MAX] [--min MIN] [-n HOST] --name NAME [-p PORT] [--session-affinity] [--volume VOLUME:/DIRECTORY_PATH] IMAGE [CMD]

用途
建立擴充群組。 依預設，Bluemix 中的容器是以分離模式執行。
參數
--anti
（選用）使用 anti-affinity，以提高容器群組的可用性。--anti 選項會強制將群組中的每個容器實例放置在個別的實體運算節點上，這會減少群組中所有容器因為硬體故障而損毀的可能性。您可能無法對較大的群組大小使用此選項，因為每一個 Bluemix 地區及組織可以部署的運算節點有限。如果部署不成功，請減少群組中的容器實例數目，或移除 --anti 選項。
--auto
（選用）若已啟用容器群組的自動回復，且群組已執行 10 分鐘，則群組的負載平衡器會透過 HTTP 要求開始定期檢查群組中每一個容器實例的性能。如果容器實例未在 100 秒內回應，則會將其標示為非作用中。非作用中容器實例會從群組中移除，並透過自動回復進行重建。自動回復會嘗試回復群組中的容器實例 3 次。在第三次嘗試之後，自動回復會有 60 分鐘的時間無法回復群組中的任何容器實例。在 60 分鐘之後，就會再次啟動自動回復處理程序。如需性能檢查的相關資訊，請參閱 --http_monitor_enabled 選項。
CMD
（選用）傳遞給容器群組執行的指令及引數。此指令必須是長時間執行指令。請不要使用短期指令（例如 /bin/date），因為這樣可能會導致容器損毀。
使用 IBM Bluemix Container Service 所提供之 ibmnode 映像檔的長時間執行指令範例：
bx ic group-create --name my_container_group registry.DomainName/ibmnode ping localhost
bx ic group-create --name my_container_group registry.DomainName/ibmnode -- tail -f /dev/null
註： CMD 及其引數必須放在 bx ic group-create 指令行尾端。
-d DOMAIN、--domain DOMAIN
（選用）預設系統網域是 AppDomainName，並且已提供 SSL 憑證，讓您可以使用 HTTPS 存取容器群組，而不需要進行任何其他配置。若要使用自訂網域，您必須在公用 DNS 伺服器上登錄自訂網域，並在 Bluemix 中配置自訂網域，然後將自訂網域對映至公用 DNS 伺服器上的 Bluemix 系統網域。 將自訂網域對映至 Bluemix 系統網域之後，該自訂網域的要求即會遞送至 Bluemix 中的應用程式。建立自訂網域時，請不要在網域名稱中包含底線 (_)。如需自訂網域的相關資訊，請參閱建立及使用自訂網域。若要讓您的自訂網域更為安全，請上傳 SSL 憑證，以使用 HTTPS 來存取您的容器群組。

合併的主機及網域可構成完整公用路徑 URL（例如 http://mycontainerhost.AppDomainName），而且在 Bluemix 內必須是唯一的。使用 bx ic group-inspect 指令來檢閱容器群組的詳細資料時，主機與網域會一起列出，作為路徑。

--desired DESIRED
（選用）群組的實例數。預設值是 2。應用程式所需要的容器群組實例數目，取決於預期的工作負載和應用程式處理要求所花的時間。例如，多工或網路連線會影響應用程式需要的實例數目。 若要判斷所要的實例數目，請在 Bluemix 中部署執行應用程式用的單一容器，並在此容器上執行負載測試。例如，如果應用程式每秒可處理 100 個要求，但您預期的平均工作負載是每秒 300 個要求，則該容器群組所要的實例數目是 3。 若要讓容器群組具備高可用性及更大的失敗復原力，請考慮包含多於最小值的額外實例來處理預期工作負載。 在另一個實例損毀時，額外實例可以處理工作負載。若要確保自動回復失敗的實例，而不影響工作負載，請包含一個額外的實例。為了預防兩個同時失敗，請包含兩個額外的實例。此設定是一種 N+2 個型樣，其中 N 是處理要求的實例數，而 +2 是額外兩個實例。
重要： 容器群組至少應該包含 2 個容器實例。如果群組中只包含一個實例，則該實例變成無法使用時，您的使用者可能會遇到應用程式關閉。
-e ENV、--env ENV
（選用）設定環境變數，其中 ENV 是 key=value 配對。個別列出多個索引鍵，而且如果包含引號，請用它們括住環境變數名稱及值。範例：-e "key1=value1" -e "key2=value2a,value2b" -e "key3=value3"

--http_monitor_enabled
（選用）預設值為 true。若已啟用容器群組的自動回復，且群組已執行 10 分鐘，則群組的負載平衡器會透過 HTTP 要求開始定期檢查群組中每一個容器實例的性能。如果容器實例未在 100 秒內回應，則會將其標示為非作用中。非作用中容器實例會從群組中移除，並透過自動回復進行重建。自動回復會嘗試回復群組中的容器實例 3 次。在第三次嘗試之後，自動回復會有 60 分鐘的時間無法回復群組中的任何容器實例。在 60 分鐘之後，就會再次啟動自動回復處理程序。 部分應用程式可能不需要進行 HTTP 性能檢查（例如，使用 TCP（而非 HTTP）進行通訊的應用程式）。若要停用 HTTP 監視器，請將此選項設為 false。範例：--http_monitor_enabled=false
--http_monitor_path
（選用）預設值為 /。啟用 --http_monitor_enabled 時，您可以使用此選項來設定進行性能檢查的自訂路徑（例如 --http_monitor_path="/health/path"）。
--http_monitor_rc_list
（選用）啟用 --http_monitor_enabled 時，您可以指定性能檢查的回應碼清單。預設值包括 200、201、202、204、300、301、302、401、403 及 404 代碼。不在有效回應清單中的任何回應，都會讓負載平衡器將容器實例的狀態變更為 INACTIVE。非作用中容器不會接收任何負載，而且容器群組的狀態會變更為 HAS_INACTIVE_INSTANCES。如果所有容器狀態都不是 ACTIVE，則容器群組的狀態會變更為 NO_ACTIVE_INSTANCES。若要進一步瞭解狀態，請參閱IBM Bluemix Container Service 中的容器狀態。
IMAGE
（必要）要包含在容器群組的每一個容器實例中的映像檔。您可在映像檔之後列出指令，但不能在映像檔之後放置任何選項；請在指定映像檔之前包含所有選項。

如果您使用組織專用 Bluemix 映像檔登錄中的映像檔，請以下列格式指定映像檔：registry.DomainName/NAMESPACE/IMAGE

如果您使用 IBM Bluemix Container Service 所提供的映像檔，請不要包含您組織的名稱空間。請以下列格式指定映像檔：registry.DomainName/IMAGE

--ip IP_ADDRESS
（選用）如果您有可用的 IP 位址，請將公用 IP 位址連結至容器群組。您也必須使用 -p 旗標來公開埠。容器群組可以有公用路徑或公用 IP 位址，但不能同時有兩者。若要取消連結 IP 位址，並將 IP 返回給空間的配額，請移除容器群組，然後重建沒有 IP 位址的群組。 若要在建立容器群組之前要求 IP 位址，請執行 bx ic ip-request。
註： 連結公用 IP 位址時不包含 SSL 憑證，這個 SSL 憑證會加密傳送至容器群組及從其中送出的資料。如果您的應用程式需要 SSL 加密，您可以實作自己的 SSL 解決方案，或將公用路徑對映至容器群組，而非連結公用 IP 位址。公用路徑已包含 SSL 憑證，讓您可以使用 HTTPS 存取容器群組，而不需要進行任何其他配置。
-m MEMORY、--memory MEMORY
（選用）輸入容器的記憶體限制 (MB)。記憶體限制是容器大小的一部分，它定義容器在執行時可在運算主機上取得的記憶體及磁碟空間數量上限。指派容器大小之後，即無法變更其值。IBM Bluemix Container Service 的可用大小如下：
  • 極微小（64 MB 記憶體，4 GB 磁碟空間）
  • 極微（128 MB 記憶體，8 GB 磁碟空間）
  • 微小（256 MB 記憶體，16 GB 磁碟空間）
  • 極小（512 MB 記憶體，32 GB 磁碟空間）
  • 小（1024 MB 記憶體，64 GB 磁碟空間）

cf ic run [-p PORT] [-P] [-d] [-e ENV] [--env-file ENVFILE] [-it] [--link NAME:ALIAS] [-m MEMORY] --name NAME [--volume VOLUME:/DIRECTORY_PATH] IMAGE [CMD [CMD ...]]

用途
在容器雲端服務中，從映像檔名稱啟動新的容器。依預設，Bluemix 中的容器是以分離模式執行。如需相關資訊，請參閱 Docker 說明中的 run 指令。
註： 您必須確定已安裝 Cloud Foundry 指令工具，而且具有 Cloud Foundry 記號。使用 cf login 及 cf ic login 順利登入後，即會產生必要的記號。
參數
CMD
（選用）傳遞給容器執行的指令及引數。此指令必須是長時間執行指令。請勿使用短期指令（例如 /bin/date），因為這樣可能會導致容器損毀。
長時間執行指令範例（假設您是使用 IBM Bluemix Container Service 登錄中提供的 ibmnode 映像檔）：
cf ic run --name my_container registry.DomainName/ibmnode ping localhost
cf ic run --name my_container registry.DomainName/ibmnode -- tail -f /dev/null
cf ic run --name my_container registry.DomainName/ibmnode -- sh -c "while true; do date; sleep 20; done"
註： CMD 及其引數必須放在 cf ic group create 指令行尾端。
-d
（選用）以分離模式執行容器。當您使用 cf ic run 時，容器預設會以分離模式執行。在分離模式下，當容器中的根程序結束時，該容器也會結束。例如，當您從 ibmnode 映像檔部署容器時，容器會在建置之後立即關閉，因為並沒有長期執行的程序。
提示： 如果您選擇使用 docker run 而非 cf ic run，則需要 -d 選項。
-e ENV、--env ENV
（選用）設定一些環境變數，其中 ENV 是 key=value 配對。個別列出多個索引鍵，而且如果包含引號，請用它們括住環境變數名稱及值。

範例

-e "key1=value1" -e "key2=value2" -e "key3=value3"

（選用）想要某個容器與另一個執行中的容器進行通訊時，即可使用主機名稱的別名予以定址。如需相關資訊，請參閱使用指令行介面 (CLI) 鏈結單一容器。
-m MEMORY、--memory MEMORY
（選用）輸入容器的記憶體限制 (MB)。記憶體限制是容器大小的一部分，它定義容器在執行時可在運算主機上取得的記憶體及磁碟空間數量上限。指派容器大小之後，即無法變更其值。IBM Bluemix Container Service 的可用大小如下：

（必要）將名稱指派給容器。

（選用）如果您要讓使用者可從網際網路存取您的應用程式，則必須公開公用埠。當您公開公用埠時，會建立容器的「公用網路安全群組」，讓您可以僅透過公開埠傳送及接收公用資料。所有其他公用埠都會關閉，因此無法用來從網際網路存取應用程式。您可以利用多個 -p 選項來包含多個埠。無法對映或轉遞埠。如果您未公開埠，則只能從專用容器網路存取容器。您可以使用已指派的專用 IP 位址來與專用網路上的容器進行通訊。

如果在 Dockerfile 中針對您要使用的映像檔指定了埠，請包含該埠。
提示：
  • 對於 IBM 認證的 Liberty Server 映像檔或此映像檔的已修改版本，請輸入埠 9080。
  • 對於 IBM 認證的 Node.js 映像檔或此映像檔的已修改版本，請輸入任何埠，例如埠 8000。

您可以選擇 UDP 與 TCP 其中一個，以指出您要使用的 IP 通訊協定。如果您未指定通訊協定，則會針對 TCP 資料流量自動公開您的埠。

範例
  • -p 9080
  • -p 9080/udp
  • -p 9080/udp -o 8000/tcp

如果空間中具有您要連結至容器的現有 IP 位址，您可以使用埠資訊指定 IP 位址。請包含 IP 位址、埠或 IP 位址與容器埠號，例如 <ip-address>:<port_on_IP>:<container-port>。IP 位址上的埠必須符合 Dockerfile 中公開的容器埠。

範例

註： 連結公用 IP 位址時不包含 SSL 憑證，這個 SSL 憑證會加密傳送至單一容器及從其中送出的資料。如果您的應用程式需要 SSL 加密，您可以實作自己的 SSL 解決方案，或使用容器群組，而非單一容器。 容器群組會連結至已包含 SSL 憑證的公用路徑，讓您可以使用 HTTPS 存取容器群組，而不需要進行任何其他配置。如需相關資訊，請參閱使用 Bluemix GUI 建立容器群組。
