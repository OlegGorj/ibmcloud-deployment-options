關於 單一及可擴充容器

IBM® Bluemix® 會在選取區中提供 IBM Bluemix Container Service 基礎架構，作為開發應用程式特性。利用容器，您可以使用任何程式設計工具，以任何語言建置應用程式。每一個容器都是隔離且安全的應用程式平台。容器可以在任何位置執行：從執行 OS X 或 Windows 的開發工作站、執行 Ubuntu 的伺服器、執行 Red Hat 的正式作業資料中心虛擬機器，或如 Bluemix 之類的雲端平台。
小心： 不提供單一及可擴充容器的免費試用。瞭解在「Bluemix 公用」上，如何在 IBM Bluemix Container Service 中進行叢集管理並移轉至 Kubernetes。

何謂容器？

容器是虛擬軟體物件，其中包含應用程式執行所需的所有元素。容器是從映像檔建置的，而此映像檔是用於建立容器的唯讀範本。每一個映像檔都只包含應用程式及其相依關係，並以主機作業系統上之隔離處理程序的形式執行。因此，容器具有資源隔離及配置的優點，但是比虛擬機器（舉例來說）更具可攜性與效率。容器可協助您快速地建置高品質的應用程式。

容器靈活多變

容器可透過為開發及正式作業小組提供標準化環境來簡化系統管理。引擎的輕量型運行環境能達到快速擴增及縮減，以回應需求變更。它們有助於消除管理不同作業系統平台及基礎架構時的複雜性。容器可協助您在任何基礎架構上快速且可靠地部署及執行任何應用程式。

容器很小

您可在單一虛擬機器所需的空間量中容納更多的容器。可以選取容器的大小，讓您只使用應用程式所需的空間量。

容器具有可攜性

使用另一個映像檔作為基礎，建置另一個容器的映像檔。讓其他人對映像檔執行大量工作，並調整映像檔以供您使用。您也可以將應用程式碼從暫置環境快速移轉至正式作業環境。您可以使用如 Delivery Pipeline 或 UrbanCode Deploy 這類的工具，來自動執行移轉處理程序。

Docker 及 IBM Bluemix Container Service

IBM Bluemix Container Service 是運用 Docker 技術的 Bluemix 運行環境。現有的 Docker 使用者以及初次使用 Docker 的使用者，可能期盼下列 IBM Bluemix Container Service 功能。

  • 將現有 Docker Hub 映像檔新增至 IBM 所管理的專用登錄。
  • 驗證映像檔以找出潛在漏洞。
  • 配置 IBM Bluemix Container Service 的指令行介面 (CLI)，以使用您已經用得順手的 Docker CLI 指令。
  • 不需要進行其他配置，即可查看容器的記載及監視資料。

如果您過去曾使用 Docker，則在開始使用 IBM Bluemix Container Service 時，可能發現這些常見問題的答案很有用。

如何在 IBM Bluemix Container Service 中使用 Docker Hub 的映像檔？
使用 IBM Bluemix Container Service CLI，您可以將映像檔從 Docker Hub 複製到專用 Bluemix 登錄。在複製映像檔之後，您可以用它來建立容器。
如何建立高度可用的容器？
IBM Bluemix Container Service 會提供與 Docker 容器類似的單一容器。此外，IBM Bluemix Container Service 提供可擴充容器群組，可以用來讓應用程式成為高度可用。容器群組提供內建的負載平衡器，以確保在容器實例之間遞送及平衡工作負載。若要提高群組的可用性，請使用反親緣性將容器群組實例分散到多台實體運算主機，並啟用性能不佳實例的自動回復。此外，您還可以設定原則，根據 CPU 及記憶體用量來自動擴充容器群組實例。如需相關資訊，請參閱將高可用程序執行為容器群組。
如何將容器埠對映至主機埠？
IBM Bluemix Container Service 會區分專用容器網路與公用網路。在專用容器網路上，預設會公開所有容器埠，而不需要進行額外配置。連接至相同專用容器網路的單一容器及容器群組，可以使用其已指派的專用 IP 位址進行通訊。不需要進行埠對映。

若要讓應用程式可供公用網路使用，您必須公開公用埠，並將公用 IP 位址連結至單一容器，或將公用路徑連結至容器群組。公開公用埠時，可讓您僅透過公開埠傳送及接收公用資料。所有其他公用埠則會予以關閉。

如何讓容器可在網際網路上公開使用？
  • 若為單一容器，請公開一個以上的埠，並將浮動 IP 位址指派給容器。
  • 若為容器群組，請公開單一 HTTP 埠，並指派一個路徑。
什麼會造成本端 Docker 與 Bluemix 的建置容器時間量不同？
您可能會注意到使用本端 Docker 安裝建置容器的速度比在 Bluemix 還要快。使用第一個部署，必須將容器映像檔下載至主機上的登錄。後續部署映像檔的速度會比較快。如需不同容器配置如何影響部署速度的相關資訊，請參閱容器類型。
如何在專用網路上建立容器，而且不公開它？
容器在建立時會連接至 IBM Bluemix Container Service 預設容器專用網路，並將專用 IP 位址指派給您的容器。在此專用網路上，預設會公開所有容器埠。不需要將主機埠對映至容器埠。若要防止可從網際網路存取容器，請不要在建立容器時連結 IP 位址或公開埠。
如何將磁區裝載至容器？
單一容器設計為短期容器。不過，您可以使用磁區在容器重新啟動之間持續保存資料。在 Docker 中裝載磁區時，會將磁區裝載到本端檔案系統。在 IBM Bluemix Container Service 中，運算主機的存取權受限，因此您無法將主機目錄裝載到容器。而是使用組織範圍的磁區，在容器重新啟動之間持續保存資料。磁區會在隔離檔案共用上管理，這些檔案共用可安全地儲存應用程式資料，以及管理對檔案的存取權和許可權。若要將磁區裝載至容器，您必須先建立磁區。從 CLI 建立容器時，請使用 --volume 選項來裝載現有磁區。從 Bluemix GUI 中，選取進階選項，並指派現有磁區。如需相關資訊，請參閱使用指令行介面 (CLI) 將檔案新增至磁區。
如何使用 Docker Compose 與 IBM Bluemix Container Service 搭配？
您可以執行 Docker Compose 指令，使用 IBM Bluemix Container Service 來建立及管理多容器部署。如需相關資訊，請參閱 指導教學：使用 Docker Compose 及 IBM Bluemix Container Service 建立多容器部署。
如何設定我的 Docker Compose 用戶端，讓它指向 IBM Containers 服務？
登入 IBM Bluemix Container Service CLI 時，請配置 CLI 來執行原生 Docker 指令。如需相關資訊，請參閱登入 bx ic 外掛程式。
如何登入我的執行中容器？要以 SSH 方式登入嗎？
您可以使用 bx ic exec 來登入容器。
為何我的容器是以分離模式執行？
依預設，Bluemix 中的容器是以分離模式執行，容許具有執行中處理程序的容器在背景執行，而不結束。以分離模式執行容器是雲端環境中的標準模型。您仍然可以透過執行 bx ic attach [--no-stdin] [--sig-proxy] CONTAINER 來附加至容器。

Bluemix 本端中 IBM Bluemix Container Service 的硬體需求

基本硬體需求
基本設定預設會配置 64 GB 記憶體的容器運算容量。
  1. 您的環境必須符合「Bluemix 本端」硬體需求。
  2. 若要使用 IBM Bluemix Container Service，每一個設定都需要有符合下列需求的控制平面。監視及記載也需要兩個實例，並符合相同的硬體需求。
    • VMware：3 部 ESX 伺服器，各自有 256 GB 記憶體及 32 個實體核心（已啟用 Hyperthreading 的 64 vCore）。跨 3 部伺服器時，需要 4 TB 的叢集共用儲存空間。
其他硬體增加的需求
最多新增 12 個擴充 VM，即可在不使用額外硬體的情況下，使用 16 GB 的增量來增加容器運算容量，最多可到 256 GB。每一個 VM 都必須有 20 GB 記憶體、4 個 vCPU 及 200 GB 磁碟空間。不過，針對叢集上部署的每 10 個擴充 VM，都需要有 1 個額外 VM 來處理網路作業。此網路作業 VM 需要 4 個 vCPU、16 GB 記憶體及 250 GB 磁碟空間。
