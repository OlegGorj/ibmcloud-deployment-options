Configuración del plug-in de IBM Bluemix Container Service (bx ic) para utilizar la CLI de Docker nativa con contenedores únicos y escalables

Instale el plug-in de IBM® Bluemix Container Service (bx ic) para ejecutar mandatos de la CLI de Docker nativa y gestionar así sus contenedores.
Atención: No hay disponibles pruebas gratuitas para los contenedores únicos y escalables. Obtenga más información acerca de la gestión de clústeres y migre a Kubernetes en IBM Bluemix Container Service en Bluemix Public.

Instalación del plug-in del servicio IBM Bluemix Container (bx ic)

Para ejecutar mandatos, el sistema operativo debe cumplir los siguientes requisitos. Esta tarea incluye la información para instalar estos requisitos. Las versiones anteriores de estas herramientas no pueden utilizarse con el plug-in de IBM Bluemix Container Service (bx ic).

  • Docker versión 1.8 a 1.12
  • Versión de CLI de Cloud Foundry 6.14.0 - 6.22.0
    Nota: Los usuarios de MacOS Sierra deben instalar la CLI de Cloud Foundry versión 6.22.0 o posterior.
  • CLI de Bluemix
  • Plug-in de IBM Bluemix Container Service

Para instalar las CLI:

  1. Instale Docker 1.12. Para los sistemas operativos Windows y OS X, Docker se instala junto con Docker Toolbox, que incluye Docker Machine, Docker Engine y Kitematic. Para los sistemas operativos Linux, los mecanismos de instalación pueden variar en función del tipo de distribución de Linux que se utilice. Consulte los siguientes enlaces para instalar o, si ya ha instalado Docker, para actualizar a Docker 1.12.
    Nota: Si ya ha instalado Docker en el sistema y no desea actualizar la versión actual, puede omitir este paso y continuar con la instalación de la CLI de Cloud Foundry.
    • OS X

      https://github.com/docker/toolbox/releases/download/v1.12/DockerToolbox-1.12.pkg

    • Windows

      https://github.com/docker/toolbox/releases/download/v1.12/DockerToolbox-1.12.exe

    • Linux

      Consulte la documentación de instalación de Docker para encontrar instrucciones sobre cómo instalar la versión de Docker 1.12 en la distribución de Linux que utiliza.

  2. Inicie Docker.
    • Para OS X y Windows, efectúe una doble pulsación en el icono Docker Quickstart Terminal. Se inicia el daemon de Docker y se establecen automáticamente las variables de entorno.
    • Para Linux, en un programa de utilidad de línea de mandatos, verifique que el daemon Docker esté en ejecución. Además, establezca el daemon de Docker de modo que se reinicie automáticamente. Para obtener más información, consulte la documentación de Docker para su sistema operativo.
  3. Verifique que la instalación de Docker se haya configurado correctamente ejecutando docker run hello-word en el terminal. 
    docker run hello-world
    Docker busca localmente la imagen hello-world desde el que ejecutar un contenedor, de modo que si esta es la primera vez que ejecuta el mandato, recibirá el siguiente mandato. 
    Unable to find image 'hello-world:latest' locally
    La imagen se obtiene de Docker Hub, un repositorio público de imágenes y, una vez completado el mandato, la salida es como la siguiente: 
    $ docker run hello-world

Hello from Docker.
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.

To try something more ambitious, you can run an Ubuntu container with:
 $ docker run -it ubuntu bash

Share images, automate workflows, and more with a free Docker Hub account:
 https://hub.docker.com

For more examples and ideas, visit:
 https://docs.docker.com/userguide/
    Si el mandato no se ha completado correctamente, ejecute docker version para ver si Docker está instalado correctamente.
  4. Instale la CLI de Cloud Foundry versión Versión de CLI de Cloud Foundry 6.14.0 - 6.22.0 desde el repositorio de GitHub. Debe instalar la CLI de Cloud Foundry en la ubicación predeterminada para el sistema operativo, de lo contrario la variable de entorno PATH no coincide con el directorio de instalación.
    Nota: Los usuarios de MacOS Sierra deben instalar la CLI de Cloud Foundry versión 6.22.0 o posterior.
    Ejecute el siguiente mandato para verificar que la CLI de Cloud Foundry se ha instalado correctamente, o si ya tenía instalada la CLI de Cloud Foundry, para comprobar qué versión ha instalado:
    cf -v
  5. Instale la CLI de Bluemix.
  6. Inicie la sesión en la CLI de Bluemix. 
    bx login -a https://api.ng.bluemix.net
  7. Instale el plug-in de IBM Bluemix Container Service. 
    bx plugin install IBM-Containers -r Bluemix
    Para comprobar que el plug-in se ha instalado correctamente, ejecute el siguiente mandato:
    bx plugin list

    El plug-in se muestra en los resultados como IBM-Containers.

Ya está preparado para iniciar una sesión y empezar a gestionar contenedores.

Inicio de sesión en el Plug-in de CLI de IBM Bluemix Container Service (bx ic)

Después de instalar la CLI, inicie sesión para empezar a utilizarla.

Antes de comenzar, complete las siguientes tareas.
  • Instale el plug-in de IBM Bluemix Container Service (bx ic).
  • Decida qué organización y espacio de Bluemix desea utilizar al iniciar la sesión en Cloud Foundry. Para obtener más información sobre las organizaciones y los espacios, consulte Organizaciones y espacios.
  • Para obtener más información sobre las opciones que puede especificar con el mandato login de Cloud Foundry, consulte el mandato Login en la documentación de Cloud Foundry.
  1. Abra un programa de utilidad de línea de mandatos.
    • Para OS X y Windows, efectúe una doble pulsación en el icono Docker Quickstart Terminal. Se inicia el daemon de Docker y se establecen automáticamente las variables de entorno.
    • Para Linux, abra cualquier programa de utilidad de línea de mandatos.
  2. Inicie sesión en Bluemix mediante la CLI de Cloud Foundry. Para especificar una región determinada, incluya la opción -a con el punto final de la API de la región. Si es la primera vez que utiliza la CLI y no incluye la opción, se le solicitará que especifique un punto final de la API. Tras especificar el punto final de la API, no es necesario indicar ninguno en los inicios de sesión posteriores. 
    bx login [-a api.DomainName] [-sso]
    Para iniciar la sesión en una región específica, incluya el punto final de API para dicha región cuando inicie la sesión en la CLI de Cloud Foundry.
    • EE.UU. Sur
      bx login -a api.ng.bluemix.net
    • Reino Unido
      bx login -a api.eu-gb.bluemix.net
    El parámetro de inicio de sesión único, --sso, se necesita cuando se inicia una sesión con un ID federado. Si se utiliza esta opción, abra https://login.ng.bluemix.net/UAALoginServerWAR/passcode para obtener una contraseña de uso único. Si no incluye la opción --sso, lleve a cabo estos subpasos.
    1. Para Correo electrónico, especifique su ID de IBM para Bluemix.
    2. Para Contraseña, especifique la contraseña para el ID de IBM. Se recuperan sus organizaciones y espacios de Bluemix.
    3. Escriba el número que represente una de sus organizaciones de Bluemix.
    4. Escriba el número que representa uno de los espacios de Bluemix existentes.
  3. Defina un espacio de nombres para la organización. Un espacio de nombres es un nombre exclusivo para identificar su registro privado de imágenes de Bluemix. Al crear un contenedor, debe especificar la ubicación de una imagen incluyendo el espacio de nombres con el nombre de la imagen.
    Nota: Cuando se crea un espacio de nombres, no se puede cambiar una vez que se ha creado, sólo se puede suprimir. Si los usuarios de la organización de Bluemix ya han desplegado contenedores y definido un espacio de nombres, aparece un mensaje que muestra el espacio de nombres de su organización. En este caso, puede continuar con el paso siguiente.
    bx ic namespace-set <my_namespace>
    Consejo: Observe las siguientes reglas del espacio de nombres:
    • Solo pueden contener sólo letras en minúscula, números o caracteres de subrayado (_).
    • Deben tener entre 4 y 30 caracteres. Si tiene intención de gestionar contenedores desde la línea de mandatos, es posible que prefiera un espacio de nombres corto que se pueda escribir rápidamente.
    • El nombre debe ser exclusivo en Bluemix.
  4. Inicialice IBM Bluemix Container Service. 
    bx ic init
  5. Decida si desea ejecutar bx ic o mandatos de Docker nativos. Puede seleccionar una de estas opciones cada vez que inicie sesión en el plug-in o bien puede utilizar el mandato shellinit para seleccionar siempre la opción 2 de forma automática.
    • Opción 1 (Valor predeterminado)

      Un shell simple para derivadores Docker de IBM Bluemix Container Service. El daemon Docker local no se ve afectado por IBM Bluemix Container Service. Los mandatos de bx ic se utilizan para gestionar contenedores en el espacio de Bluemix y los mandatos docker se utilizan para gestionar contenedores de forma local. Los mandatos docker son siempre necesarios para el desarrollo de imágenes local.

      Por ejemplo, con la Opción 1, el mandato bx ic run crea un contenedor en el espacio de Bluemix y docker run crea un contenedor de forma local. Es posible que prefiera esta opción si no está familiarizado con Docker ni IBM Bluemix Container Service o si desea probar los contenedores de forma local.

    • Opción 2

      Redirija el daemon Docker local para la sesión de CLI actual para IBM Bluemix Container Service, de modo que los mandatos docker se puedan utilizar en lugar de los mandatos bx ic siempre que estén disponibles para gestionar contenedores en su espacio de Bluemix.

      Por ejemplo, con la opción 2, tanto el mandato bx ic run como el mandato docker run se pueden utilizar para crear un contenedor en su espacio. Es posible que prefiera esta opción si es un usuario familiarizado con Docker que está más cómodo con los mandatos docker o si está adaptando los scripts automatizados para trabajar con IBM Bluemix Container Service.

      Hay algunas excepciones que todavía requieren el uso de los mandatos bx ic para efectuar tareas específicas de IBM Bluemix Container Service. Por ejemplo, para establecer un espacio de nombres, debe utilizar el mandato bx ic namespace-set namespace-set. docker namespace-set no es un mandato válido. Para obtener una lista de mandatos docker a los que se da soporte con esta configuración, consulte Mandatos de Docker admitidos para el plug-in de IBM Bluemix Container Service (bx ic).

      Para configurar esta opción, copie y pegue las variables de entorno que se proporcionan en el terminal durante el inicio de sesión.
      • DOCKER_HOST establece el host de Docker en IBM Bluemix Container Service.
      • DOCKER_CERT_PATH establece la ubicación de las claves de autenticación del plug-in de IBM Bluemix Container Service para Cloud Foundry.
      • DOCKER_TLS_VERIFY utiliza TLS y verifica el servidor remoto.
      Consejo: Cuando complete las tareas en la documentación de IBM Bluemix Container Service, podrá ejecutar los mandatos docker en todos los pasos marcados con un asterisco (*).

Actualización del plug-in de IBM Bluemix Container Service (bx ic)

Actualice el plug-in periódicamente para utilizar las nuevas funciones.

Tabla 1. Versión actual del plug-in de IBM Bluemix Container Service
Fecha de lanzamiento Versión Actualizaciones
16 de noviembre de 2016 0.8.964
  • Se ha añadido soporte para gestionar la comprobación de estado para los grupos de contenedores mediante el mandato bx ic group-create utilizando las opciones --http_monitor_enabled, --http_monitor_path y --http_monitor_rs_list
  • Adición de soporte para la opción bx ic group-create --session_affinity
  • Arreglos de defectos

Para actualizar el plug-in, siga estos pasos:

  1. Instale la actualización desde el repositorio de plug-ins que ha configurado al instalar el plug-in por primera vez. 
    bx plugin update IBM-Containers -r Bluemix
  2. Para verificar la instalación del plug-in, ejecute el mandato siguiente y compruebe la lista de plug-ins instalados. 
    bx plugin list

    El plug-in se muestra en los resultados como IBM-Containers.

Desinstalación del plug-in de IBM Bluemix Container Service (bx ic)

Si ya no necesita el plug-in de IBM Bluemix Container Service, puede desinstalarlo con el mandato bx.

Nota: Para desinstalar el plug-in de IBM Bluemix Container Service Cloud Foundry, ejecute el siguiente mandato. 
cf uninstall-plugin IBM-Containers
  1. Para desinstalar el plug-in de IBM Bluemix Container Service para la CLI de Bluemix, ejecute el siguiente mandato. 
    bx plugin uninstall IBM-Containers
  2. Para verificar la desinstalación del plug-in, ejecute el mandato siguiente y compruebe la lista de plug-ins de instalados. 
    bx plugin list

    El plug-in de IBM Bluemix Container Service no se muestra en los resultados.

Instalación de Docker Compose y sus dependencias

Para iniciar y detener varios contenedores únicos simultáneamente, puede instalar la CLI de Docker Compose.

Para ejecutar mandatos de Docker Compose con IBM Bluemix Container Service, debe tener instalado el siguiente software.
  • Docker versión 1.10.0
  • CLI de Cloud Foundry versión 6.14.0 - 6.22.0
  • CLI de Bluemix
  • Plug-in de IBM Bluemix Container Service
  • Docker Compose versión 1.6.0 o posterior
Esta tarea incluye la información para instalar estos requisitos. Las versiones anteriores de estas herramientas no pueden utilizarse con el plug-in de IBM Bluemix Container Service (bx ic).
  1. Instale Docker, la CLI de Cloud Foundry, la CLI de Bluemix y el plug-in de IBM Bluemix Container Service.
    Nota: Si utiliza un sistema operativo Mac o Windows, Docker Compose se instala automáticamente con Docker Engine, Machine y Kitematic como parte de la instalación de Docker Toolbox. Puede omitir el siguiente paso y seguir verificando la versión de Docker Compose.
  2. Si está utilizando Linux, instale la última versión de Docker Compose.
    1. Instale pip. El gestor de paquetes pip incluye todas las herramientas para instalar, actualizar y eliminar paquetes de software que se escriben en Python en el sistema. Puede utilizar el siguiente mandato para instalar pip o puede descargar el paquete de software directamente en el sistema desde Python Package Index.
      Nota: Si ya ha instalado pip en el sistema, puede continuar con la instalación de la última versión de Docker Compose. 
      easy_install pip
    2. Instale Docker Compose 
      pip install docker-compose==1.6.2
  3. Verifique la versión Docker Compose que está instalada en el sistema.
    1. Inicie la sesión en la CLI de IBM Bluemix Container Service. Establezca las variables de entorno para Opción 2 cuando inicie la sesión en el plug-in para ejecutar mandatos Docker nativos.
    2. Verifique la versión del paquete de Docker Compose. 
      docker-compose version
      Si se visualiza docker-compose version 1.6.0 o posterior, empiece a crear su propia app de varios contenedores. Si no, actualice su versión de Docker Compose.
      Nota: Si tiene problemas al ejecutar los mandatos de Docker Compose, revise el tema Ningún módulo llamado cola y Aviso de conexión HTTPS insegura para comprobar si necesita cambiar la configuración del sistema o instalar paquetes de software adicionales.

Actualización de Docker Compose

Es posible que necesite actualizar la versión del paquete de Docker Compose para utilizar las últimas características.

  1. Actualice Docker Compose. 
    pip install docker-compose --upgrade
  2. Verifique la versión del paquete de Docker Compose. 
    docker-compose version

Desinstalación de Docker Compose

Si ya no necesita el paquete de software de Docker Compose, puede desinstalarlo con el siguiente mandato.

  1. Desinstale Docker Compose. 
    pip uninstall docker-compose
  2. Verifique que el paquete de Docker Compose se ha desinstalado. Si Docker Compose se ha desinstalado satisfactoriamente, no se lista en la salida de la CLI cuando ejecuta el siguiente mandato. 
    pip freeze
