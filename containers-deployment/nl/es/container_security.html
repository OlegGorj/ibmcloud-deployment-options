Seguridad para contenedores únicos y escalables

IBM® Bluemix Container Service ofrece una plataforma en la nube segura y de confianza que optimiza las prestaciones de seguridad de Linux y Docker. Al crear y ejecutar contenedores con IBM Bluemix Container Service, su app se beneficia del aislamiento del contenedor y la red, de acceso limitado a los recursos y datos de host, y de los despliegues seguros de contenedor.
Atención: No hay disponibles pruebas gratuitas para los contenedores únicos y escalables. Obtenga más información acerca de la gestión de clústeres y migre a Kubernetes en IBM Bluemix Container Service en Bluemix Public.

Valores de seguridad en el host de cálculo de IBM Bluemix Container Service

IBM Bluemix Container Service optimiza las prestaciones de seguridad de Docker y el kernel de Linux para garantizar el aislamiento del contenedor y restringir el acceso a recursos de host de cálculo.

Daemons de Docker totalmente gestionados y protegidos
Los daemons de Docker que se ejecutan en los hosts de cálculo de IBM se configuran sin ningún acceso directo de usuarios y sólo los puede configurar IBM. Todos los sockets de daemon de Docker están protegidos con certificados TLS.
Espacios de nombres de usuario de Docker habilitados
La característica de espacio de nombres de usuario de Docker está habilitada en el Docker Engine de los hosts de cálculo. La característica de espacio de nombres de usuario aísla los procesos que se ejecutan dentro de un contenedor de todos los demás procesos y contenedores que se ejecutan en el mismo host de cálculo. El usuario root efectivo del contenedor se asigna a un usuario no root del host de cálculo. Incluso aunque un usuario root del contenedor pueda descomponer el contenedor, el usuario no tendrá permisos para manipular archivos, procesos u otros contenedores que se ejecutan en el host de cálculo.
Contenedores con privilegios no permitidos
No se da soporte a la opción --privileged del mandato run. Los contenedores no tienen acceso a dispositivos ni discos duros del host de cálculo.
Grupos de control gestionados (cgroups) para memoria, CPU y espacio de disco
Los contenedores que se ejecutan en el mismo host de cálculo comparten memoria, CPU y espacio de disco. Para impedir que un contenedor consuma todos los recursos disponibles o que apague aplicaciones o sistemas host, la cantidad máxima de memoria, CPU y espacio de disco que se pueden asignar a un contenedor es limitada.

La cantidad de memoria, CPU y espacio de disco disponible para el contenedor durante el tiempo de ejecución está determinada por el tamaño del contenedor que seleccione durante la creación del contenedor. Esta limitación garantiza que cada contenedor del host de cálculo obtenga su parte correcta de los recursos de host disponibles. Si un contenedor comienza a fallar, el límite de recursos garantiza un rendimiento y un comportamiento coherentes para los otros contenedores del host de cálculo.

Optimización del kernel de host de cálculo
Los kernels de host de cálculo limitan el número total de hebras y procesos que se ejecutan en el host de cálculo y que pueden ejecutar un grupo o usuario específico. Esta optimización garantiza que el host no esté sobrecargado y no se quede sin ranuras de proceso hasta que se alcance el máximo del usuario.
Supervisión de host de cálculo continuo
Cada host de cálculo está continuamente supervisado por IBM para controlar y solucionar bombas fork y otros ataques de denegación de servicio (Denial-Of-Service, DOS) de nivel de proceso.
Perfil de AppArmor habilitado para contenedores de Docker
Cada host de cálculo se aplica al perfil de AppArmor específico de IBM para proteger el entorno y restringir las posibilidades de un contenedor en el host de cálculo. AppArmor es un módulo de seguridad de kernel de Linux que controla el acceso a carpetas, archivos, dominios de red, así como los permisos para crear y cambiar datos.
Montaje de volúmenes local inhabilitado en los hosts de cálculo
IBM Bluemix Container Service no da soporte al montaje de volúmenes locales o directorios locales/host en un contenedor. En su lugar, los volúmenes de ámbito de la organización se utilizan para la persistencia de datos entre los reinicios del contenedor y para que los datos estén disponibles para varios contenedores. Los volúmenes se alojan en comparticiones de archivos aisladas que almacenan de forma segura datos de apps y gestionan el acceso y el permiso a los archivos.

Los volúmenes de ámbito de la organización deben crearse antes de montarlos en el contenedor. Puede crear un volumen desde la GUI o la CLI de Bluemix.

Valores de seguridad en la red del contenedor privada de IBM Bluemix Container Service

Revise la lista de funciones de IBM Bluemix Container Service integradas que garantizan la comunicación segura entre contenedores de la misma red privada, entre contenedores de distintos espacios de Bluemix, y entre el espacio de Bluemix y una red remota.

Red de contenedor aislada de L2 privada por espacio
Una red privada de contenedor crea un entorno aislado y seguro para los contenedores únicos y los grupos de contenedores que se ejecutan en un espacio. Los contenedores que están conectados a la misma red privada pueden enviar y recibir datos de otros contenedores en la red privada, utilizando las direcciones IP privadas. Los contenedores no están públicamente disponibles hasta que se enlazan un puerto público y una dirección IP pública para contenedores únicos o una ruta pública para grupos de contenedores.

En Bluemix, cada espacio se proporciona con una red privada de contenedor donde se aplican los valores de red de IBM Bluemix Container Service predeterminados. Al crear contenedores, estos se conectan automáticamente a la red privada predeterminada del espacio y se les asigna una dirección IP privada.

Mantenga los contenedores privados o expóngalos selectivamente servicios destinados al público a través de los grupos de seguridad de red
Cada contenedor está protegido por grupos de seguridad de red específicos de IBM que controlan el tráfico de red entrante y sirven como cortafuegos de software en el nivel de pasarela de red privada. Se comprueba la conformidad de los datos que se envían a través de la pasarela de red privada respecto a las reglas de grupos de seguridad de red y se pueden pasar al contenedor de destinatarios o bloquearse.

Cuando se crea un contenedor y no se expone públicamente, se configura con el grupo de seguridad de red privada predeterminado. Este grupo de seguridad permite la comunicación de red privada entre contenedores usando las direcciones IP privadas. De forma predeterminada, todos los puertos de contenedor se exponen automáticamente en la red privada. No tiene que correlacionar los puertos de contenedor a un puerto de host.

Para convertir su app en públicamente disponible, debe exponer un puerto público y enlazar una dirección IP pública a su contenedor único, o una ruta pública a su grupo de contenedores. Al exponer un puerto público mediante la opción -p en el mandato bx ic run, crea un grupo de seguridad de red pública para el contenedor que permite que los datos públicos se envíen y reciban solo en el puerto expuesto. Los demás puertos públicos están cerrados y no pueden utilizarse para acceder a la app desde Internet.

Supervisión de red de empresa continua
Todo el tráfico de red de IBM Bluemix Container Service lo supervisa IBM para detectar y solucionar actividades maliciosas.
Soporte completo para los mandatos exec, attach y logs de Docker
Puede iniciar sesión en el contenedor o ver su salida detallada durante el tiempo de ejecución aprovechando los mandatos bx ic exec, bx ic attach o bx ic logs. Con estos mandatos, puede evitar ejecutar un daemon SSH no seguro para acceder y supervisar el contenedor durante el tiempo de ejecución.

Para obtener más información sobre cómo iniciar sesión de forma segura en el contenedor mediante bx ic exec, consulte Inicio de sesión en un contenedor con exec.

VPN para el acceso de la red privada segura a los servicios de otros espacios y en cualquier red remota
Los contenedores de una red de contenedores privada se pueden conectar de forma segura a centros de datos de la empresa mediante el servicio IBM® Virtual Private Network.

Para obtener más información, consulte el apartado sobre Configuración de una conexión VPN entre los contenedores y un centro de datos corporativo

Inicio de sesión en un contenedor con exec

Si debe iniciar la sesión en el contenedor de ejecución, puede utilizar bx ic exec.

Los usos comunes del mandato bx ic exec incluyen la apertura de una ventana bash y la visualización de archivos de un volumen asignado a un contenedor. El mandato bx ic exec solamente se ejecuta en un contenedor que se encuentre en estado en ejecución. Para mantener un contenedor en ejecución, este debe iniciarse con un mandato de larga ejecución. Si un contenedor se pone en pausa o se detiene, debe reiniciarse con un mandato de larga ejecución para poder iniciar sesión en él.

Mandato y parámetros

bx ic exec [-d] [-it ] [-u USER] CONTAINER CMD
Nota: *En este mandato, puede sustituir bx ic con docker al iniciar sesión en IBM Bluemix Container Service y establecer las variables de entorno para utilizar mandatos Docker nativos.

Ejemplos

Este mandato de ejemplo ejecuta un terminal bash interactivo en el contenedor my_container.
bx ic exec -it my_container bash
Este mandato de ejemplo ejecuta el mandato date en el contenedor my_container. 
bx ic exec my_container date

Conexión de contenedores a un centro de datos corporativo configurando una red privada virtual

Conectar de forma segura los contenedores únicos y los grupos de contenedores en una red de contenedores privada en Bluemix con un centro de datos corporativo utilizando el servicio IBM® Virtual Private Network (VPN). IBM VPN proporciona un canal de comunicaciones de extremo a extremo seguro sobre internet que está basado en la suite de protocolos Internet Protocol Security (IPsec) estándar del sector. El protocolo IPsec ofrece la autenticación de iguales a nivel de red, la integridad de datos y la confidencialidad de datos cifrando los paquetes que se intercambian entre los puntos finales VPN. Para configurar una conexión segura entre los contenedores en Bluemix y un centro de datos corporativo, debe tener instalados una pasarela VPN IPsec o un servidor SoftLayer en el centro de datos local. Con el servicio de IBM VPN, puede configurar una pasarela VPN por espacio y definir hasta 16 conexiones en distintos destinos.

Para conectar contenedores a un centro de datos corporativo, siga estos pasos.

  1. Cree al menos un contenedor único o grupo de contenedores en el espacio que desea conectar con el centro de datos corporativo.
  2. Cree una instancia del servicio de IBM VPN en el espacio siguiendo los pasos 1 a 5 del tema Enlace de un servicio.
    Nota: La instancia de servicio de IBM VPN debe crearse en el mismo espacio en el que ha creado los contenedores que desea conectar con el centro de datos corporativo.
  3. Siga las instrucciones del tema Iniciación a IBM VPN para configurar la pasarela VPN y configurar conexiones con su centro de datos de la empresa.
