Revisión de vulnerabilidades de contenedores

En el panel de control, puede ver el estado de un contenedor para determinar si su seguridad cumple con la política de la organización. También puede revisar el informe de vulnerabilidad de un contenedor, que detalla los paquetes vulnerables y los valores de aplicaciones o de contenedores no seguros.

Compruebe que los contenedores que se ejecutan en el espacio siguen cumpliendo con la política de organización revisando el campo POLICY STATUS. El estado se muestra como una de las siguientes condiciones:
  • Aprobado: no se han encontrado vulnerabilidades.
  • Aviso: Vulnerability Advisor ha encontrado vulnerabilidades potenciales en el contenedor que han provocado que el contenedor no cumpla con la política. Si la política de la organización permite el despliegue de imágenes vulnerables, es posible que la imagen se haya desplegado en el estado (Desplegar con cuidado), y que se envíe un aviso al usuario que la ha desplegado.
  • Infracciones de política: Vulnerability Advisor ha encontrado vulnerabilidades potenciales en el contenedor que han provocado que el contenedor no cumpla con la política. La política de la organización está establecida para no permitir el despliegue de las imágenes vulnerables.
  • Incompleto: la exploración no se ha completado. La exploración puede seguir en ejecución, o el sistema operativo de la instancia del contenedor puede no ser compatible.

Si el ESTADO DE POLÍTICA no es Aprobado, compruebe que el contenedor sea lo más seguro posible visualizando su informe de vulnerabilidad y actuando en cualquier vulnerabilidad de la que se haya informado, realizando los pasos siguientes:

  1. En el panel de control de Bluemix, seleccione el contenedor que desee revisar.
  2. En el panel Vulnerability Advisor, pulse Ver informe.
  3. Revise las secciones para ver las vulnerabilidades potenciales para cada paquete en la imagen:
    • Políticas de la organización

      Lista si el contenedor cumple con la política establecida por el gestor de la organización.

    • Paquetes vulnerables

      Lista los paquetes con problemas de vulnerabilidad conocidos, actualizados diariamente a partir de avisos de seguridad publicados para los sistemas operativos listados en Comprobación de vulnerabilidades de imágenes utilizando el Vulnerability Advisor. Por lo general, para que un paquete vulnerable pase la exploración, se necesita una versión posterior del paquete que incluya una corrección para la vulnerabilidad en cuestión. El mismo paquete puede tener varias vulnerabilidades; en tal caso, una única actualización del paquete puede corregir varios problemas. Pulse el código de aviso de seguridad para revisar más información sobre el paquete y para ver los pasos para actualizar el paquete.

    • Valores de contenedor

      Lista sugerencias que puede adoptar para aumentar la seguridad del contenedor. La columna Acción de corrección describe cómo mejorar la seguridad.

    • Configuraciones de aplicación

      Lista valores de aplicaciones para el contenedor que no son seguros. La columna Acción de corrección describe cómo mejorar la seguridad.

    • Origen de imagen de Docker

      Muestra la imagen que se ha utilizado para crear este contenedor.

    Para cada elemento mostrado, se suministran acciones de corrección o sugerencias.

  4. Decida cómo actualizar el contenedor para que pueda resolver los problemas. Asegúrese de comprender bien la arquitectura del contenedor para elegir el método apropiado de actualización del contenedor. Por ejemplo:
    • Si el contenedor está desacoplado de los datos que calcula, puede detener el contenedor, realizar los cambios necesarios a la imagen, y volver a desplegar.
    • Puede utilizar Active Deploy para actualizar la instancia de contenedor vulnerable sin detener el contenedor.
    • En una configuración de HA con varios contenedores en un grupo de contenedores, puede direccionar tráfico a otra instancia de contenedor mientras arregla las vulnerabilidades.
  5. Solucione los problemas descritos en el informe Valoración de la vulnerabilidad y vuelva a crear la imagen o vuelva a desplegar el contenedor de acuerdo con el método que seleccione. Algunos problemas del Dockerfile se pueden resolver usando el código suministrado en Resolución de vulnerabilidades en contenedores o imágenes.
