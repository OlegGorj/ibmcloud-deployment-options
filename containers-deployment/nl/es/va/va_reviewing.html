Revisión de un informe de VA de imágenes

Antes de desplegar una imagen, puede revisar su informe de Vulnerability Advisor, que detalla los paquetes vulnerables, el contenedor no seguro o los valores de aplicaciones, y si la imagen cumple con las políticas de la organización.

Las imágenes del contenedor las proporciona IBMo las puede añadir la organización. Es posible que tenga que configurar una imagen para poder desplegarla de modo que satisfaga los requisitos establecidos por la política de la organización. Sólo el gestor de organización puede cambiar la política para permitir que se despliegue una imagen vulnerable. Para obtener los pasos para ver y cambiar la directiva de la organización, consulte Revisión de las políticas de la organización.

Revise los problemas potenciales de configuración y seguridad completando los siguientes pasos:

  1. Inicie una sesión en Bluemix. Vulnerability Advisor esta disponible en la interfaz de usuario solo cuando haya iniciado sesión.
  2. En la sección Valoración de la vulnerabilidad, revise el estado de la valoración de la vulnerabilidad. El estado se muestra como una de las siguientes condiciones:
    • Seguro para desplegar: no se han encontrado problemas de seguridad.
    • Desplegar con cuidado: se han encontrado problemas de seguridad o de configuración que hay que solucionar, y la política organizativa permite el despliegue de imágenes vulnerables. Si despliega esta imagen, recibirá un aviso.
    • Despliegue bloqueado: se han encontrado problemas de seguridad o de configuración que deberían corregirse para poder desplegar la imagen porque la política organizativa no permite el despliegue de imágenes vulnerables.
    • Incompleto: la exploración no se ha completado. La exploración puede seguir en ejecución o el sistema operativo de la imagen puede no ser compatible. Espere y vuelva a intentar la exploración de nuevo. Si la exploración no se ha completado aún, envíe por push la imagen de nuevo para iniciar una nueva exploración. Las imágenes con exploraciones incompletas no se bloquean para el despliegue.

    Revise el informe detallado que forma la evaluación de seguridad de Vulnerability Advisor:

  3. Seleccione la imagen y en la sección Valoración de la vulnerabilidad, pulse Ver informe.
  4. Revise las secciones para ver los problemas potenciales de seguridad y de configuración para cada paquete de la imagen:
    • Políticas de la organización

      Lista si la imagen cumple con la política establecida por el gestor de la organización.

    • Análisis de riesgos

      Si la imagen tiene una o varias vulnerabilidades, esta sección muestra la tasa de riesgo calculada por Vulnerability Advisor. El riesgo se calcula mediante el calculador de CVSS y la tecnología IBM® X-Force Exchange.

    • Paquetes vulnerables

      Lista los paquetes con problemas de vulnerabilidad conocidos, que se actualizan a diario a partir de avisos de seguridad publicados para los tipos de imagen de Docker que se muestran en la lista de Gestión de la seguridad de los contenedores y de las imágenes con Vulnerability Advisor. Por lo general, para que un paquete vulnerable pase la exploración, se necesita una versión posterior del paquete que incluya una corrección para la vulnerabilidad en cuestión. El mismo paquete puede tener varias vulnerabilidades; en tal caso, una única actualización del paquete puede corregir varios problemas. Pulse el código de aviso de seguridad para revisar más información sobre el paquete y para ver los pasos para actualizar el paquete.

    • Valores de contenedor

      Lista sugerencias que puede adoptar para aumentar la seguridad de la imagen. La columna Acción de corrección describe cómo mejorar la seguridad.

    • Configuraciones de aplicación

      Lista valores de aplicaciones para la imagen que no son seguros. La columna Acción de corrección describe cómo mejorar la seguridad.

    • Instancias de contenedor

      Lista contenedores que utilizan esta imagen que ya están en ejecución en su organización.

    Para cada elemento mostrado, se suministran acciones de corrección o sugerencias.

  5. Solucione los problemas descritos en el informe Valoración de la vulnerabilidad y vuelva a crear la imagen. Algunos problemas del Dockerfile se pueden resolver usando el código suministrado en Resolución de vulnerabilidades en contenedores o imágenes.
Tras desplegar la imagen, Vulnerability Advisor seguirá buscando problemas de seguridad y de configuración en el contenedor. Puede resolver cualquier problema encontrado siguiendo los pasos que se describen en Revisión de vulnerabilidades de contenedores.
