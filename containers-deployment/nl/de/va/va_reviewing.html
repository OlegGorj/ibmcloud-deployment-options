Image-VA-Bericht überprüfen

Bevor Sie ein Image bereitstellen, können Sie den zugehörigen Vulnerability Advisor-Bericht prüfen, in dem Sie Informationen zu gefährdeten Paketen und zu nicht sicheren Container- oder Anwendungseinstellungen sowie Angaben dazu finden, ob das Image mit den geltenden Organisationsrichtlinien konform ist.

Container-Images werden von IBMzur Verfügung gestellt oder sie können von Ihrer Organisation hinzugefügt werden. Möglicherweise müssen Sie ein Image konfigurieren, bevor Sie es bereitstellen können, damit es den Anforderungen entspricht, die durch die Richtlinie Ihrer Organisation vorgegeben werden. Nur der Organisationsmanager kann die Richtlinie so ändern, dass ein gefährdetes Image bereitgestellt werden kann. Die Schritte zum Anzeigen und Ändern der Organisationsrichtlinie sind im Abschnitt Organisationsrichtlinien überprüfen beschrieben.

Prüfen Sie potenzielle Sicherheits- und Konfigurationsprobleme bei Images, indem Sie die folgenden Schritte ausführen:

  1. Melden Sie sich bei Bluemix an. Die Vulnerability Advisor (Vulnerability Assessment) ist nur in der Benutzerschnittstelle verfügbar, wenn Sie angemeldet sind.
  2. Prüfen Sie im Abschnitt Schwachstellenanalyse den Status der Schwachstellenanalyse (Vulnerability Assessment). Der Status wird als eine der folgenden Bedingungen angezeigt:
    • Sicher für Bereitstellung - Keine Sicherheitsprobleme gefunden.
    • Mit Vorsicht bereitstellen - Es wurden Sicherheits- oder Konfigurationsprobleme gefunden, die behoben werden sollten, und die Organisationsrichtlinie lässt die Bereitstellung gefährdeter Images zu. Wenn Sie dieses Image bereitstellen, empfangen Sie eine Warnung.
    • Bereitstellung geblockt - Es wurden Sicherheits- oder Konfigurationsprobleme gefunden, die behoben werden müssen, bevor das Image bereitgestellt werden kann, weil Ihre Organisationsrichtlinie die Bereitstellung gefährdeter Images nicht zulässt.
    • Unvollständig - Der Scan ist nicht vollständig. Möglicherweise wird der Scanvorgang noch ausgeführt oder das Betriebssystem des Image ist nicht kompatibel. Warten Sie und versuchen Sie erneut, den Scan auszuführen. Wenn der Scanvorgang auch nach einer Wartezeit nicht abgeschlossen ist, wiederholen Sie die Push-Operation für das Image, um einen neuen Scan zu starten. Images mit unvollständigen Scans werden nicht für die Bereitstellung blockiert.

    Informieren Sie sich anhand des detaillierten Berichts, der die Vulnerability Advisor-Sicherheitsbewertung enthält:

  3. Wählen Sie das Image aus und klicken Sie im Abschnitt Schwachstellenanalyse auf Bericht anzeigen.
  4. Überprüfen Sie die folgenden Abschnitte, um potenzielle Sicherheits- und Konfigurationsprobleme für jedes Paket im Image zu ermitteln:
    • Organisationsrichtlinien

      Gibt an, ob das Image der vom Organisationsmanager festgelegten Richtlinie entspricht.

    • Risikoanalyse

      Weist das Image eine oder mehrere Sicherheitslücken auf, wird in diesem Abschnitt die Risikobewertung wie durch Vulnerability Advisor berechnet angezeigt. Die Berechnung des Risikos erfolgt wird mit der CVSS-Berechnungsfunktion unter Verwendung von IBM® X-Force Exchange-Technologie.

    • Gefährdete Pakete

      Listet Pakete mit bekannten Problemen im Hinblick auf Sicherheitslücken auf. Die Liste dieser Pakete wird täglich auf Basis veröffentlichter Sicherheitshinweise für die Docker-Image-Typen in Sicherheit für Container und Images mit Vulnerability Advisor verwalten aktualisiert. In der Regel wird zur erfolgreichen Durchführung eines Scans für ein gefährdetes Paket eine aktuellere Version des Pakets benötigt, das eine Korrektur für die Sicherheitslücke umfasst. Für ein Paket können mehrere Sicherheitslücken aufgelistet werden; in einem solchen Fall können durch ein einzelnes Upgrade für ein Paket mehrere Probleme behoben werden. Klicken Sie auf den Sicherheitshinweiscode, um weitere Informationen zum Paket und zu den Schritten zu erhalten, mit denen Sie das Paket aktualisieren.

    • Containereinstellungen

      Listet Empfehlungen für Maßnahmen auf, mit denen die Sicherheit des Image erhöht werden kann. In der Spalte mit den Maßnahmen zur Problembehebung wird die Vorgehensweise zum Verbessern der Sicherheit beschrieben.

    • Anwendungskonfigurationen

      Listet Anwendungseinstellungen für das Image auf, die nicht sicher sind. In der Spalte mit den Maßnahmen zur Problembehebung wird die Vorgehensweise zum Verbessern der Sicherheit beschrieben.

    • Containerinstanzen

      Listet Container auf, die bereits in Ihrer Organisation aktiv sind.

    Es werden Maßnahmen und Empfehlungen zur Problembehebung für jede aufgelistete Position angegeben.

  5. Beheben Sie die im Bericht zur Schwachstellenanalyse beschriebenen Probleme und erstellen Sie das Image neu. Einige Probleme in der Dockerfile können behoben werden, indem der im Abschnitt Sicherheitslücken in Containern oder Images beheben angegebene Code verwendet wird.
Wenn Sie Ihr Image bereitgestellt haben, setzt Vulnerability Advisor das Scannen auf Sicherheits- und Konfigurationsprobleme im Container fort. Sie können gefundene Probleme beheben, indem Sie die Schritte ausführen, die unter Sicherheitslücken bei Containern überprüfen beschrieben sind.
