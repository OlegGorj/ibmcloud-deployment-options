Sicherheitslücken bei Containern überprüfen

Im Dashboard können Sie anhand der Statusangabe für einen Container feststellen, ob dessen Sicherheit den Richtlinien Ihrer Organisation entspricht. Sie können auch den Bericht zu Sicherheitslücken eines Containers überprüfen, der Schwachstellen bei Paketen und Einstellungen zu nicht sicheren Containern oder Anwendungen aufzeigt.

Stellen Sie sicher, dass in Ihrem Bereich aktive Container weiterhin den Organisationsrichtlinien entsprechen, indem Sie das Feld POLICY STATUS überprüfen. Der Status wird als eine der folgenden Bedingungen angezeigt:
  • Bestanden - Keine Schwachstellen gefunden.
  • Warnung - Vulnerability Advisor hat potenzielle Schwachstellen in Ihrem Container gefunden, die dazu führen, dass der Container nicht der Richtlinie entspricht. Wenn Ihre Organisationsrichtlinie die Bereitstellung gefährdeter Images zulässt, wurde das Image möglicherweise mit dem Status Mit Vorsicht bereitstellen bereitgestellt und es wurde eine Warnung an den Benutzer gesendet, der es zur Verfügung gestellt hat.
  • Richtlinienverstoß - Vulnerability Advisor hat potenzielle Schwachstellen in Ihrem Container gefunden, die dazu führen, dass der Container nicht der Richtlinie entspricht. Ihre Organisationsrichtlinie lässt die Bereitstellung gefährdeter Images nicht zu.
  • Unvollständig - Der Scan ist nicht vollständig. Möglicherweise wird der Scanvorgang noch ausgeführt oder das Betriebssystem dieser Containerinstanz ist nicht kompatibel.

Wenn der Richtlinienstatus nicht Bestanden lautet, prüfen Sie, dass der Container so sicher wie möglich ist, indem Sie den entsprechenden Bericht zu den Sicherheitslücken anzeigen und durch das Ausführen der folgenden Schritte auf gemeldete Sicherheitslücken reagieren:

  1. Wählen Sie im Bluemix-Dashboard den Namen des Containers aus, der überprüft werden soll.
  2. Klicken Sie im Bereich Vulnerability Advisor auf Bericht anzeigen.
  3. Überprüfen Sie die folgenden Abschnitte, um potenzielle Sicherheitslücken für jedes Paket im Image zu ermitteln:
    • Organisationsrichtlinien

      Gibt an, ob der Container der vom Organisationsmanager festgelegten Richtlinie entspricht.

    • Gefährdete Pakete

      Listet Pakete mit bekannten Problemen im Hinblick auf Sicherheitslücken auf. Die Liste wird täglich anhand von veröffentlichten Sicherheitshinweisen für die in Sicherheitslücken bei Images mit Vulnerability Advisor überprüfen aufgeführten Betriebssysteme aktualisiert. In der Regel wird zur erfolgreichen Durchführung eines Scans für ein gefährdetes Paket eine aktuellere Version des Pakets benötigt, das eine Korrektur für die Sicherheitslücke umfasst. Für ein Paket können mehrere Sicherheitslücken aufgelistet werden; in einem solchen Fall können durch ein einzelnes Upgrade für ein Paket mehrere Probleme behoben werden. Klicken Sie auf den Sicherheitshinweiscode, um weitere Informationen zum Paket und zu den Schritten zu erhalten, mit denen Sie das Paket aktualisieren.

    • Containereinstellungen

      Listet Empfehlungen für Maßnahmen auf, mit denen die Sicherheit des Containers erhöht werden kann. In der Spalte mit den Maßnahmen zur Problembehebung wird die Vorgehensweise zum Verbessern der Sicherheit beschrieben.

    • Anwendungskonfigurationen

      Listet Anwendungseinstellungen für den Container auf, die nicht sicher sind. In der Spalte mit den Maßnahmen zur Problembehebung wird die Vorgehensweise zum Verbessern der Sicherheit beschrieben.

    • Docker-Image-Quelle

      Zeigt das Image an, das zum Erstellen dieses Containers verwendet wurde.

    Es werden Maßnahmen und Empfehlungen zur Problembehebung für jede aufgelistete Position angegeben.

  4. Entscheiden Sie sich, wie der Container aktualisiert werden soll, damit Sie die Probleme beheben können. Stellen Sie sicher, dass Sie gut mit der Containerarchitektur vertraut sind, damit Sie eine geeignete Methode für die Containeraktualisierung auswählen können. Beispiel:
    • Wenn der Container von den von ihm verarbeiteten Daten entkoppelt wird, können Sie den Container stoppen, die erforderlichen Änderungen am Image vornehmen und den Container erneut bereitstellen.
    • Mit Active Deploy können Sie die gefährdeten Containerinstanzen aktualisieren, ohne den Container zu stoppen.
    • In einer Hochverfügbarkeitskonfiguration mit mehreren Containern in einer Containergruppe können Sie beim Beheben der Sicherheitslücken den Datenverkehr an eine andere Containerinstanz weiterleiten.
  5. Beheben Sie die im Bericht zur Schwachstellenanalyse beschriebenen Probleme und erstellen Sie das Image neu oder stellen Sie den Container mit der von Ihnen gewählten Methode erneut bereit. Einige Probleme in der Dockerfile können behoben werden, indem der im Abschnitt Sicherheitslücken in Containern oder Images beheben angegebene Code verwendet wird.
