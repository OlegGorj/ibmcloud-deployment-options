Informationen zu einzelnen und skalierbaren Containern

IBM® Bluemix bietet die IBM Bluemix Container Service-Infrastruktur in ausgewählten Regionen als Feature für die App-Entwicklung an. Mit Containern können Sie Ihre App ganz ohne Programmierungstool in jeder beliebigen Sprache erstellen. Jeder Container ist eine isolierte und sichere App-Plattform. Ein Container kann überall ausgeführt werden: Auf Entwicklungsworkstations, auf denen OS X oder Windows ausgeführt wird, auf Servern mit Ubuntu, auf virtuellen Maschinen in Produktionsrechenzentren, auf denen Red Hat ausgeführt wird, oder auf einer Cloudplattform wie Bluemix.
Achtung: Kostenlose Testversionen für einzelne und skalierbare Container sind nicht verfügbar. Lesen Sie die Informationen zum Cluster-Management und zur Migration auf Kubernetes in IBM Bluemix Container Service unter Bluemix Public.

Was ist ein Container?

Container sind virtuelle Softwareobjekte, die alle Elemente enthalten, die eine App für die Ausführung benötigt. Ein Container wird aus einem Image erstellt, bei dem es sich um eine schreibgeschützte Schablone zum Erstellen des Containers handelt. Jedes einzelne Image enthält nur die App mit den zugehörigen Abhängigkeiten und wird als isolierter Prozess auf dem Hostbetriebssystem ausgeführt. Daher verfügt ein Container über die Vorteile der Isolation und Zuordnung von Ressourcen, ist jedoch weitaus portierbarer und effizienter als zum Beispiel eine virtuelle Maschine. Mit Containern können schnell leistungsstarke Anwendungen erstellt werden.

Container sind agil

Container vereinfachen die Systemverwaltung durch die Bereitstellung standardisierter Umgebungen für Entwicklungs- und Produktionsteams. Die schlanke Laufzeitumgebung der Engine ermöglicht schnelle Scale-up- und Scale-down-Operationen als Antwort auf Änderungen in der Nachfrage. Sie helfen dabei, die Verwaltung unterschiedlicher Betriebssystemplattformen und der zugrunde liegenden Infrastruktur zu erleichtern. Mithilfe von Containern können Sie jede App in jeder Infrastruktur bereitstellen und schnell und zuverlässig ausführen.

Container sind klein

Sie können in dem Speicherbereich, den eine einzelne virtuelle Maschine benötigen würde, mehrere Container unterbringen. Die Größe eines Containers kann ausgewählt werden, sodass Sie nur den Speicherbereich nutzen, den die App benötigt.

Container sind portierbar

Erstellen Sie ein Image für einen Container, indem Sie ein anderes Image als Grundlage verwenden. So wird die Hauptarbeit für ein Image von jemand anderem ausgeführt und Sie müssen das Image nur noch für Ihre Nutzung optimieren. Sie können auch App-Code schnell von einer Staging-Umgebung in eine Produktionsumgebung migrieren. Der Migrationsprozess kann mit Tools wie Delivery Pipeline oder UrbanCode Deploy automatisiert werden.

Docker und IBM Bluemix Container Service

Bei IBM Bluemix Container Service handelt es sich um eine Bluemix-Laufzeitumgebung, die das Potenzial der Docker-Technologie nutzt. Erfahrene wie neue Docker-Benutzer werden die im Folgenden beschriebenen Möglichkeiten, die IBM Bluemix Container Service bietet, zu schätzen wissen.

  • Fügen Sie Ihr vorhandenes Docker Hub-Image in eine private Registry ein, die von IBM gehostet wird.
  • Validieren Sie Images im Hinblick auf potenzielle Sicherheitslücken.
  • Konfigurieren Sie die Befehlszeilenschnittstelle (CLI) für IBM Bluemix Container Service für eine Verwendung der in Docker verfügbaren CLI-Befehle, mit denen Sie gerne arbeiten.
  • Zeigen Sie Protokollierungs- und Überwachungsdaten für Ihre Container ohne zusätzliche Konfigurationsmaßnahmen an.

Wenn Sie Docker in der Vergangenheit verwendet haben, können Ihnen die Antworten auf die folgenden häufig gestellte Fragen weiterhelfen, wenn Sie mit der Verwendung von IBM Bluemix Container Service beginnen.

Wie kann ein Docker Hub-Image in IBM Bluemix Container Service verwendet werden?
Über die IBM Bluemix Container Service-CLI können Sie ein Docker Hub-Image in Ihre private Bluemix-Registry kopieren. Nachdem Sie das Image kopiert haben, können Sie damit einen Container erstellen.
Wie kann ein Container mit hoher Verfügbarkeit erstellt werden?
IBM Bluemix Container Service stellt einzelne Container bereit, die mit Docker-Containern vergleichbar sind. Darüber hinaus enthält IBM Bluemix Container Service skalierbare Containergruppen, mit deren Hilfe Ihre Anwendung hoch verfügbar gemacht werden kann. Containergruppen enthalten eine integrierte Lastausgleichsfunktion, die sicherstellt, dass die Workload gleichmäßig auf die Containerinstanzen verteilt wird. Verteilen Sie Ihre Containergruppeninstanzen auf mehrere physische Rechenhosts mit Anti-Affinität und aktivieren Sie die automatische Wiederherstellung von Instanzen, die nicht ordnungsgemäß ausgeführt werden. Darüber hinaus können Sie Richtlinien für das automatische Skalieren von Containergruppeninstanzen gemäß der CPU- und Speicherbelegung festlegen. Weitere Informationen finden Sie unter Hoch verfügbare Prozesse als Containergruppen ausführen.
Wie werden Container-Ports zu einem Host-Port zugeordnet?
Bei IBM Bluemix Container Service wird zwischen dem privaten Containernetz und dem öffentlichen Netz unterschieden. Im privaten Containernetz sind alle Container-Ports standardmäßig ohne zusätzliche Konfiguration zugänglich. Einzelne Container und Containergruppen, die mit demselben privaten Containernetz verbunden sind, können unter Verwendung ihrer zugewiesenen privaten IP-Adressen kommunizieren. Eine Zuordnung von Ports ist nicht erforderlich.

Sie können Ihre App im öffentlichen Netz zur Verfügung stellen, indem Sie einen öffentlichen Port zugänglich machen und entweder eine öffentliche IP-Adresse an Ihren einzelnen Container oder eine öffentliche Route an Ihre Containergruppe binden. Wenn ein öffentlicher Port zugänglich gemacht wird, ermöglicht dies das ausschließliche Empfangen und Senden von öffentlichen Daten über den zugänglich gemachten Port. Alle sonstigen öffentlichen Ports werden geschlossen.

Wie kann ein Container im Internet öffentlich zugänglich gemacht werden?
  • Stellen Sie für einzelne Container mindestens einen Port zur Verfügung und ordnen Sie dem Container eine variable IP-Adresse zu.
  • Stellen Sie für Containergruppen einen einzelnen HTTP-Port bereit und ordnen Sie eine Route zu.
Wodurch entsteht der unterschiedliche Zeitaufwand für die Erstellung eines Containers bei der lokalen Docker-Installation und Bluemix?
Möglicherweise stellen Sie fest, dass Container bei der lokalen Docker-Installation schneller als in Bluemix erstellt werden. Bei der ersten Bereitstellung muss das Container-Image in die Registry auf dem Host heruntergeladen werden. Nachfolgende Bereitstellungen des Image werden schneller ausgeführt. Weitere Informationen zum Einfluss der unterschiedlichen Containerkonfigurationen auf die Geschwindigkeit der Bereitstellung finden Sie unter Containertypen.
Wie kann ein Container in einem privaten Netz erstellt und nicht öffentlich zugänglich gemacht werden?
Wenn Sie einen Container erstellen, wird er mit dem privaten IBM Bluemix Container Service-Standardcontainernetz verbunden und dem Container wird eine private IP-Adresse zugewiesen. In diesem privaten Netz sind alle Container-Ports standardmäßig zugänglich. Eine Zuordnung von Host-Ports zu Container-Ports ist nicht erforderlich. Wenn der Zugriff auf einen Container über das Internet verhindert werden soll, darf bei der Erstellung des Containers keine IP-Adresse gebunden bzw. kein Port bereitgestellt werden.
Wie kann ein Datenträger an einen Container angehängt werden?
Ein einzelner Container ist designbedingt kurzlebig. Mit einem Datenträger können Sie jedoch Daten zwischen Containerneustarts persistent speichern. Wenn Sie einen Datenträger in Docker anhängen, wird der Datenträger an Ihr lokales Dateisystem angehängt. In IBM Bluemix Container Service ist der Zugriff auf den Rechenhost eingeschränkt, sodass Sie Hostverzeichnisse nicht an einen Container anhängen können. Stattdessen werden organisationsbezogene Datenträger verwendet, um Daten zwischen Containerneustarts zu erhalten. Datenträger werden auf isolierten Dateifreigaben gehostet, die App-Daten sicher speichern und den Dateizugriff und Dateiberechtigungen verwalten. Zum Anhängen eines Datenträgers an einen Container müssen Sie zunächst einen Datenträger erstellen. Hängen Sie beim Erstellen eines Containers über die Befehlszeilenschnittstelle einen vorhandenen Datenträger mit der Option --volume an. Wählen Sie in der Bluemix-GUI Erweiterte Optionen aus und weisen Sie einen vorhandenen Datenträger zu. Weitere Informationen finden Sie unter Dateien über die Befehlszeilenschnittstelle (CLI) zu Datenträgern hinzufügen.
Wie kann Docker Compose mit IBM Bluemix Container Service verwendet werden?
Docker Compose-Befehle können ausgeführt werden, um Multi-Container-Bereitstellungen mit IBM Bluemix Container Service zu erstellen und zu verwalten. Weitere Informationen finden Sie unter Lernprogramm: Multi-Container-Bereitstellung mit Docker Compose und IBM Bluemix Container Service erstellen.
Wie kann der Docker Compose-Client eingerichtet werden, damit er auf den IBM Containers-Service verweist?
Wenn Sie sich bei der IBM Bluemix Container Service-Befehlszeilenschnittstelle anmelden, konfigurieren Sie die Befehlszeilenschnittstelle für die Ausführung nativer Docker-Befehle. Weitere Informationen enthält Beim bx ic-Plug-in anmelden.
Wie erfolgt die Anmeldung an einem aktiven Container?
Sie können sich mit bx ic exec bei einem Container anmelden.
Warum werden meine Container im Modus mit aufgehobener Zuordnung ausgeführt?
Standardmäßig werden Container in Bluemix im Modus mit aufgehobener Zuordnung ausgeführt, damit die Ausführung von Containern mit aktiven Prozessen ohne Unterbrechung im Hintergrund fortgesetzt werden kann. Die Ausführung von Containern im Modus mit aufgehobener Zuordnung entspricht dem Standardmodell für eine Cloudumgebung. Durch Ausführen von bx ic attach [--no-stdin] [--sig-proxy] CONTAINER können Sie trotzdem eine Verbindung zu Ihrem Container herstellen.

IBM Bluemix Container Service steht in Bluemix Dedicated und Bluemix Local zur Verfügung.

Hardwarevoraussetzungen für IBM Bluemix Container Service in Bluemix Local

Basishardwarevoraussetzungen
Die Basiseinrichtung wird standardmäßig mit 64 GB Speicher für die Berechnungskapazität für Container konfiguriert.
  1. Ihre Umgebung muss die Hardwarevoraussetzung für Bluemix Local erfüllen.
  2. Um IBM Bluemix Container Service verwenden zu können, ist für jede Installation eine Control Plane erforderlich, die der folgenden Voraussetzungen erfüllt. Zur Überwachung und Protokollierung werden außerdem zwei Instanzen benötigt, die die gleichen Hardwarevoraussetzungen erfüllen.
    • VMware: Drei ESX-Server, jeder mit 256 GB Speicher und 32 physischen Cores (64 vCore mit aktiviertem Hyperthreading). Auf diesen drei Servern ist 4 TB gemeinsam genutzter Speicher für das Clustering erforderlich.
Voraussetzungen für zusätzliche Hardwareerweiterungen
Sie können die Containerberechnungskapazität in 16 GB Schritten bis 256 GB ohne zusätzlich erforderliche Hardware bis auf 12 Erweiterungs-VMs erhöhen. Jede VM muss 20 GB Speicher, 4 vCPU und 200 GB Plattenspeicherplatz aufweisen. Für jede 10 im Cluster bereitgestellten Erweiterungs-VMs ist eine zusätzliche VM für den Netzbetrieb erforderlich. Diese VM für den Netzbetrieb benötigt 4 vCPU, 16 GB Speicher und 250 GB Plattenspeicherplatz.
