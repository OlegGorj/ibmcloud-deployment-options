Persistente Daten in einem Datenträger für einzelne und skalierbare Container speichern

Ein Speicherdatenträger dient dazu, Daten im Bluemix-Bereich einer Organisation zu speichern oder Containern für die gemeinsame Nutzung zur Verfügung zu stellen.
Achtung: Kostenlose Testversionen für einzelne und skalierbare Container sind nicht verfügbar. Lesen Sie die Informationen zum Cluster-Management und zur Migration auf Kubernetes in IBM® Bluemix Container Service unter Bluemix Public.
Diese Seite enthält folgende Abschnitte:
     • Informationen zu Datenträgern
     • Datenträger über die GUI oder die CLI erstellen
     • Dateien zu Datenträgern hinzufügen
     • Datenträger sichern und wiederherstellen
     • Datenträgerzugriff für Benutzer ohne Rootberechtigung hinzufügen
     • Daten auf anderes Rechenzentrum migrieren
     • Datenträger entfernen
     • Dateifreigaben hinzufügen und entfernen

Informationen zu Datenträgern

Ein Container ist kein dauerhaftes Element, sodass Anwendungsdaten dort nicht persistent gespeichert werden können. Mit einem Datenträger ist es jedoch möglich, Daten zwischen Containerneustarts persistent zu speichern, Containerdaten in einem Bereich zur gemeinsamen Nutzung bereitzustellen und auf die Anwendungsdaten zuzugreifen, wenn der Container nicht verfügbar ist. Wenn Sie einen Container löschen, wird der Speicherdatenträger nicht gelöscht. Der Lebenszyklus eines Speicherdatenträgers ist unabhängig vom Lebenszyklus eines Containers. Da von Docker-Containern in Bluemix nicht auf den lokalen Host zugegriffen werden kann, werden die Datenträger an bereitgestellte Dateifreigaben angehängt.

Der Lebenszyklus für einen persistenten Speicher beinhaltet die folgenden Tasks:

  1. Optional: Erstellen Sie über die GUI oder die CLI eine angepasste Dateifreigabe, die Ihre Datenträger bereitstellt. Sie müssen die Rolle eines Organisationsmanagers haben, um eine Dateifreigabe erstellen zu können. Wenn Sie kein Organisationsmanager sind, fahren Sie mit dem Erstellen eines Datenträgers fort. Für jede Bluemix-Organisation wird standardmäßig eine Dateifreigabe von 20 GB bereitgestellt.
  2. Erstellen Sie einen Datenträger über die GUI oder die CLI.
  3. Erstellen Sie eine automatische Sicherung (Backup) Ihres Datenträgers mit dem Image 'ibm-backup-restore'.
  4. Löschen Sie einen Datenträger.
  5. Löschen Sie eine Dateifreigabe.

Datenträger mit der Bluemix-GUI erstellen

Ein Datenträger ist eine persistente Speicherposition für die Daten, die eine App erstellt oder die Dateien, die die App für die Ausführung erfordert. Über die Bluemix-GUI können Sie einen Datenträger für Ihren Container erstellen.

Ein einzelner Container ist designbedingt kurzlebig. Mit einem Datenträger können Sie jedoch Daten zwischen Containerneustarts persistent speichern, Daten zwischen den Containern in einem Bereich gemeinsam nutzen und Daten zwischen den Containerinstanzen in einer Gruppe gemeinsam nutzen. Wenn Sie einen Datenträger in Docker anhängen, wird der Datenträger an Ihr lokales Dateisystem angehängt. In IBM Bluemix Container Service ist der Zugriff auf den Rechenhost eingeschränkt, sodass Sie Hostverzeichnisse nicht an einen Container anhängen können. Stattdessen werden organisationsbezogene Datenträger verwendet, um Daten zwischen Containerneustarts zu erhalten. Datenträger werden auf isolierten Dateifreigaben gehostet, die App-Daten sicher speichern und den Dateizugriff und Dateiberechtigungen verwalten. Aktualisierungen, die Sie an einem Container oder Image vornehmen, wirken sich nicht auf einen Datenträger aus. Wenn Sie einen Container löschen, wird der Speicherdatenträger nicht gelöscht.

  1. Wählen Sie im Katalog Container und anschließend ein Image aus.
  2. Wählen Sie Erweiterte Optionen aus, um die Felder für die Datenträgererstellung zu erweitern.
  3. Klicken Sie auf Datenträger erstellen.
  4. Geben Sie einen Namen in das Feld Datenträgername ein. Datenträger sind pro Bereich eindeutig. Daher können Datenträgernamen innerhalb eines Bereichs nicht wiederverwendet werden. Der Datenträgername darf Großbuchstaben, Kleinbuchstaben, Ziffern, Unterstreichungszeichen (_) und Bindestriche (-) enthalten.
  5. Wenn eine Dateifreigabe vorhanden ist, wählen Sie im Dropdown-Menü Dateifreigabe eine vorhandene Dateifreigabe aus. Wenn keine Dateifreigabe vorhanden ist, wird automatisch eine Standarddateifreigabe von 20 GB erstellt. Der Organisationsmanager kann Dateifreigaben mit bestimmten Speichergrößen und IOPS-Werten zuordnen. Weitere Informationen finden Sie unter Dateifreigaben für Datenträger hinzufügen. Für Datenträger gelten keine Größenbeschränkungen. Datenträger können bis zu der Größe der Dateifreigabe skaliert werden, auf der sie gehostet werden.
  6. Klicken Sie auf Erstellen. Die Erstellung des Datenträgers kann einige Minuten dauern.
Der Datenträger ist nun in Ihrem Bereich verfügbar und kann an Ihren einzelnen Container oder Ihre Containergruppe angehängt werden, indem Sie ihn im Dropdown-Menü über Vorhandenen Datenträger zuweisen unter Erweiterte Optionen auswählen. Nachdem Sie den Datenträger ausgewählt haben, weisen Sie den Dateipfad zu, an den der Datenträger innerhalb des Containers angehängt wird, wie z. B. /var/lib/my_volume. Falls Sie eine Sicherung (Backup) Ihres Containers benötigen, lesen Sie die Informationen in Daten sichern und wiederherstellen.

Datenträger über die Befehlszeile (CLI) erstellen

Ein Datenträger ist eine persistente Speicherposition für die Daten, die eine App erstellt oder die Dateien, die die App für die Ausführung erfordert. Über die Befehlszeile (CLI) können Sie einen Datenträger für Ihren Container erstellen.

Ein einzelner Container ist designbedingt kurzlebig. Mit einem Datenträger können Sie jedoch Daten zwischen Containerneustarts persistent speichern, Daten zwischen den Containern in einem Bereich gemeinsam nutzen und Daten zwischen den Containerinstanzen in einer Gruppe gemeinsam nutzen. Wenn Sie einen Datenträger in Docker anhängen, wird der Datenträger an Ihr lokales Dateisystem angehängt. In IBM Bluemix Container Service ist der Zugriff auf den Rechenhost eingeschränkt, sodass Sie Hostverzeichnisse nicht an einen Container anhängen können. Stattdessen werden organisationsbezogene Datenträger verwendet, um Daten zwischen Containerneustarts zu erhalten. Datenträger werden auf isolierten Dateifreigaben gehostet, die App-Daten sicher speichern und den Dateizugriff und Dateiberechtigungen verwalten. Aktualisierungen, die Sie an einem Container oder Image vornehmen, wirken sich nicht auf einen Datenträger aus. Wenn Sie einen Container löschen, wird der Speicherdatenträger nicht gelöscht.

  1. Erstellen Sie einen Datenträger, indem Sie den folgenden Befehl ausführen. Datenträger können bis zu der Größe der Dateifreigabe skaliert werden, auf der sie gehostet werden. Die Angabe einer Dateifreigabe ist optional. Sie können bestehende Dateifreigaben mit dem Befehl bx ic volume-fs anzeigen. Wenn keine Dateifreigabe vorhanden ist, wird automatisch eine Standarddateifreigabe von 20 GB erstellt. Der Organisationsmanager kann Dateifreigaben mit bestimmten Speichergrößen und IOPS-Werten zuordnen. Weitere Informationen finden Sie unter Dateifreigaben für Datenträger hinzufügen. 
    bx ic volume-create VOLNAME [FSNAME]
    bx ic volume-create-Parameter
    FSNAME
    (Erforderlich) Der Name der Dateifreigabe. Wenn keine Dateifreigabe verfügbar ist oder benannt wurde, wird zur Erstellung des Datenträgers die Standarddateifreigabe des Bereichs verwendet.
    VOLNAME
    (Erforderlich) Der Name des Datenträgers. Der Name darf Großbuchstaben, Kleinbuchstaben, Ziffern, Unterstreichungszeichen (_) und Bindestriche (-) enthalten.
  2. Optional: Vergewissern Sie sich mit dem folgenden Befehl, dass der Datenträger ordnungsgemäß erstellt wurde. 
    bx ic volumes [--table]

Als Nächstes weisen Sie den Datenträger Ihrem einzelnen Container oder Ihrer Containergruppe zu. Datenträger werden mit der Option --volume im Befehl run bzw. im Befehl group-create zugewiesen. Falls Sie eine Sicherung (Backup) Ihres Containers benötigen, lesen Sie die Informationen in Daten sichern und wiederherstellen.

Dateien über die Befehlszeilenschnittstelle (CLI) zu Datenträgern hinzufügen

Nach der Erstellung eines Datenträgers können Sie den Datenträger an einen Container anhängen und Dateien hinzufügen, auf die alle Container zugreifen können, an die der Datenträger angehängt ist.

Erstellen Sie mit der Bluemix-GUI oder der Befehlszeilenschnittstelle (CLI) einen Datenträger, bevor Sie mit dem Folgenden fortfahren.
  1. Erstellen Sie einen Container mit einem angehängten Datenträger.* Ersetzen Sie mein_datenträger durch den Namen Ihres Datenträgers, containerpfad durch den Mountpfad des Datenträgers, mein_container durch den Namen des Containers und 'registry.DomainName/mein_namensbereich/mein_image' durch das Image. 
    bx ic run --volume mein_datenträger:/containerpfad --name mein_container registry.DomainName/<mein_namensbereich>/mein_image
    Tipp: Führen Sie bx ic namespace-get aus, um Ihren Namensbereich abzurufen, und ersetzen Sie <mein_namensbereich> durch Ihren Namensbereich.
    Anmerkung: *In diesem Befehl können Sie bx ic durch docker ersetzen, wenn Sie bei IBM Bluemix Container Service angemeldet sind und Ihre Umgebungsvariablen zur Verwendung nativer Docker-Befehle definiert haben. Sie können native Docker-Befehle in allen Schritten verwenden, die in diesem Abschnitt mit einem Stern (*) gekennzeichnet sind.
  2. Kopieren Sie mit dem Befehl bx ic cp Dateien aus einem lokalen Verzeichnis auf den Datenträger.*

    In diesem Beispiel kopieren Sie eine Datei config.txt auf Ihren Datenträger, der im Container an den Pfad /src/usr/tmp angehängt ist. Ersetzen Sie die Datei durch Ihre eigene Datei bzw. Ihr eigenes Verzeichnis, die/das Sie kopieren und ändern Sie den Containerpfad so, dass er auf Ihren Datenträger verweist. In dem Befehl wird die Quellendatei bzw. das Ursprungsverzeichnis als Erstes und der Zielpfad als Zweites angegeben. Wenn Sie in einen oder aus einem Container kopieren, müssen Sie den Namen des Containers, das Begrenzungszeichen ":" und den Pfad zu der Zieldatei bzw. zum Zielordner einschließen. Der Pfad muss relativ zum Stammverzeichnis des Containers angegeben werden. Weitere Information enthält die Dokumentation zu bx ic cp.

    bx ic cp ./config.txt mein_container:/src/usr/tmp
  3. Bestätigen Sie, dass Ihre Dateien auf den Datenträger kopiert wurden. Melden Sie sich dazu mit dem Befehl bx ic exec bei Ihrem aktiven Container an.* 
    bx ic exec -it mein_container bash
  4. Navigieren Sie zu dem Verzeichnis, an das Sie Ihren Datenträger angehängt haben, und listen Sie die Dateien auf. Ihre Datei wird in der Ausgabe aufgelistet. Alle Container, die an diesen Datenträger angehängt sind, haben Zugriff auf diese Datei. 
    # cd containerpfad
# ls
config.txt

Daten über die Befehlszeilenschnittstelle (CLI) sichern und wiederherstellen

Containerdatenträger werden im persistenten Speicher gehostet, Backups sind jedoch keine verfügbar. Falls Sie eine Sicherung (Backup) Ihrer Datenträgerdaten benötigen, können Sie mit bx ic cp eine manuelle Sicherung erstellen.

  1. Ermitteln Sie den Pfad des angehängten Datenträgers für den aktiven oder gestoppten Container, für den Sie eine Sicherung ausführen wollen.* 
    bx ic inspect mein_container

    Die Ausgabe enthält einen Abschnitt ähnlich dem folgenden Beispiel mit dem Namen für den Datenträger und dem Mountpunkt des Containers. In diesem Beispiel ist der Ordner /mnt/data der Mountpunkt. 

            "Mounts": [
            {
                "Destination": "/mnt/data",
                "Driver": "",
                "Mode": "rw",
                "Name": "",
                "RW": true,
                "Source": "/vol/fa277ff4-8a64-435b-9b75-0f11d59a3ae4/beispiel_container-id"
            }
    Anmerkung: *In diesem Befehl können Sie bx ic durch docker ersetzen, wenn Sie bei IBM Bluemix Container Service angemeldet sind und Ihre Umgebungsvariablen zur Verwendung nativer Docker-Befehle definiert haben. Sie können native Docker-Befehle in allen Schritten verwenden, die in diesem Abschnitt mit einem Stern (*) gekennzeichnet sind.
  2. Optional: Stoppen Sie alle aktiven Container, die zu diesem Zeitpunkt Daten auf den Datenträger schreiben.* Durch das Stoppen der Container verhindern Sie, dass Daten kopiert werden, für die ein Schreibvorgang ausgeführt wird. 
    bx ic stop mein_container
  3. Kopieren Sie die Dateien aus dem einzelnen Container auf Ihren lokalen Computer.* Wenn Sie einen Datenträger für eine Containergruppe sichern wollen, reicht es, diesen Befehl auf nur einer Instanz der Gruppe auszuführen, da ja alle Instanzen denselben Mountpunkt mit dem Datenträger gemeinsam nutzen.

    Sie besitzen nun eine lokale Sicherung (Backup) ihrer Datenträgerdaten. 

    bx ic cp mein_container:/mnt/data mein_lokaler_ordner
    Tabelle 1. Elemente des Befehls 'bx ic cp'
    Erklärung der Bestandteile dieses Befehls
    cp Der Befehl zum Kopieren.
    mein_container:/mnt/data An erster Stelle steht der Quellenpfad. In diesem Fall kopieren Sie die Daten Ihres Containers an ein lokales Ziel. Daher ist der Containerdateipfad die Quelle. Sie müssen den Namen des Containers gefolgt von einem Doppelpunkt und dem Dateipfad innerhalb des Containers einschließen. Im vorliegenden Beispiel haben Sie den Mountpunkt des Datenträgers aus dem vorherigen Schritt ausgewählt, sodass Sie alle enthaltenen Dateien kopieren können.
    mein_lokaler_ordner An zweiter Stelle steht der Zielpfad. Der lokale Pfad kann als absoluter Pfad oder als relativer Pfad angegeben werden. In diesem Beispiel kopieren Sie die Daten Ihres Containers in den relativen Pfad mein_lokaler_ordner.
  4. Optional: Kopieren Sie Ihre Daten aus dem lokalen Ordner auf einen neuen Datenträger.* Dieser Befehl bewirkt die Wiederherstellung Ihrer Daten in einem neuen Container oder auf einem neuen Datenträger. Ein Grund, der für eine Übertragung von Daten auf einen neuen Datenträger spricht, wäre der Umstand, dass dieser Datenträger auf einer neu erstellten Dateifreigabe angehängt ist, die ihrerseits über mehr Speicher und einen höheren IOPS-Wert verfügt als die alte Dateifreigabe. Informationen zum Erstellen einer neuen Dateifreigabe enthält Dateifreigaben über die Befehlszeilenschnittstelle (CLI) hinzufügen. Informationen zum Erstellen eines neuen Datenträgers finden Sie unter Datenträger über die Befehlszeile (CLI) erstellen. 
    bx ic cp mein_lokaler_ordner mein_neuer_container:/mnt/data
Ihre Daten werden nun in Ihren lokalen Ordner kopiert und Sie verfügen über eine manuelle Sicherung. Außerdem haben Sie die Daten auf einen Datenträger auf einem neuen Container kopiert.

Datenträgerzugriff für Benutzer ohne Rootberechtigung über die Befehlszeilenschnittstelle (CLI) hinzufügen

Benutzer ohne Rootberechtigung verfügen nicht über die Schreibberechtigung für den Mountpfad für Datenträger. Um die Schreibberechtigung zu erteilen, müssen Sie die Dockerfile des Image bearbeiten und im Mountpfad ein Verzeichnis mit der entsprechenden Berechtigung zu erstellen.

Bei IBM Bluemix Container Service ist die Funktion für Benutzernamensbereiche für Docker Engine aktiviert. Für Benutzernamensbereiche wird eine Isolationsfunktion bereitgestellt, d. h. der Rootbenutzer kann weder auf andere Container noch auf den Rechenhost zugreifen. Datenträger befinden sich jedoch auf externen NFS-Dateifreigaben außerhalb der Rechenhosts für Container. Die Datenträger sind so konfiguriert, dass der Rootbenutzer im Container erkannt wird. Da sich die Datenträger jedoch außerhalb des Containers befinden, können sie die Benutzernamensbereiche nicht erkennen.

Bei einigen Anwendungen ist nur der Rootbenutzer innerhalb des Containers registriert. In vielen Anwendungen wird jedoch ein Benutzer ohne Rootberechtigung angegeben, der in den Container-Mountpfad schreibt. Wenn Sie eine Anwendung erstellen, in der ein Benutzer ohne Rootberechtigung die Schreibberechtigung für den Datenträger benötigt, müssen Sie die folgenden Prozesse in Ihrer Dockerfile und in Ihrem Script für den Einstiegspunkt hinzufügen:

  • Einen Benutzer ohne Rootberechtigung erstellen
  • Den Benutzer temporär zu Rootgruppe hinzufügen
  • Ein Verzeichnis mit den entsprechenden Benutzerberechtigungen im Datenträger-Mountpfad erstellen

  1. Erstellen Sie eine Dockerfile in einem lokalen Verzeichnis. Die nachfolgende Beispiel-Dockerfile erstellt einen Benutzer ohne Rootberechtigung mit dem Namen myguest. 
    FROM registry.domain_name/ibmnode:latest

#Gruppe und Benutzer mit Gruppen-ID & Benutzer-ID 1010 erstellen.
#In diesem Fall wird eine Gruppe und ein Benutzer mit dem Namen myguest erstellt.
#Die Gruppen-ID und die Benutzer-ID 1010 verursacht wahrscheinlich keinen Konflikt mit bestehenden GUIDs oder UIDs in dem Image.
#Die Gruppen-ID und Benutzer-ID muss im Bereich von 0 bis 65536 liegen. Andernfalls schlägt die Containererstellung fehl.
RUN groupadd --gid 1010 myguest
RUN useradd --uid 1010 --gid 1010 -m --shell /bin/bash myguest

ENV MY_USER=myguest

COPY entrypoint.sh /sbin/entrypoint.sh
RUN chmod 755 /sbin/entrypoint.sh

EXPOSE 22
ENTRYPOINT ["/sbin/entrypoint.sh"]
  2. Erstellen Sie das Einstiegspunktscript im selben lokalen Ordner wie die Dockerfile. In diesem Beispiel wird im Einstiegspunktscript der Datenträger-Mountpfad /mnt/myvol angegeben. 
    #!/bin/bash
set -e

#Dies ist der Mountpunkt für den gemeinsam genutzten Datenträger.
#Verwenden Sie diesen Mountpunkt mit dem Ausführungsbefehl 'bx ic'.
#Standardmäßig ist der Rootbenutzer Eigner des Mountpunkts.
MOUNTPATH="/mnt/myvol"
MY_USER=${MY_USER:-"myguest"}

#Diese Funktion erstellt ein Unterverzeichnis, dessen Eigner
#der Benutzer ohne Rootberechtigung im Mountpfad des gemeinsamen Datenträgers ist.
create_data_dir() {
  #Den Benutzer ohne Rootberechtigung zur Primärgruppe für Rootbenutzer hinzufügen.
  usermod -aG root $MY_USER

  #Der Gruppe Lese-, Schreib- und Ausführungsberechtigung für den Mountpfad des gemeinsamen Datenträgers erteilen.
  chmod 775 $MOUNTPATH
  #Im gemeinsamen Pfad des Eigners myguest ohne Rootberechtigung ein Verzeichnis erstellen.
  su -c "mkdir -p ${MOUNTPATH}/mydata" -l $MY_USER
  su -c "chmod 700 ${MOUNTPATH}/mydata" -l $MY_USER
  ls -al ${MOUNTPATH}

  #Den Benutzer ohne Rootberechtigung aus Sicherheitsgründen aus der Rootbenutzergruppe entfernen.
  deluser $MY_USER root

  #Die ursprünglichen Lese-, Schreib- und Ausführungsberechtigungen für den Mountpfad des Datenträgers wiederherstellen.
  chmod 755 $MOUNTPATH
  echo "Datenverzeichnis wurde erstellt..."
}

create_data_dir

#Dieser Befehl erstellt einen Prozess mit langer Laufzeit für dieses Beispiel.
tail -F /dev/null
  3. Melden Sie sich bei IBM Bluemix an. 
    bx login [-a api.DomainName] [-sso]
    Der Parameter --sso für Single Sign-on ist erforderlich, wenn Sie sich mit einer eingebundenen ID anmelden. Rufen Sie bei Verwendung dieser Option https://login.ng.bluemix.net/UAALoginServerWAR/passcode auf, um einen nur für eine einzige Anmeldung gültigen Kenncode abzurufen. Führen Sie die folgenden untergeordneten Schritte aus, wenn Sie die Option --sso nicht angeben:
    1. Geben Sie für Email Ihre IBM ID für Bluemix an.
    2. Geben Sie für Kennwort das Kennwort für die IBM ID an. Daraufhin werden die Bluemix-Organisationen und -Bereiche abgerufen.
    3. Geben Sie die Nummer für eine Ihrer Bluemix-Organisationen ein.
    4. Geben Sie die Nummer für einen Ihrer bestehenden Bluemix-Bereiche an.
  4. Initialisieren Sie die IBM Bluemix Container Service-CLI. 
    bx ic init
  5. Fügen Sie das Image zu Ihrer Bluemix-Registry hinzu.* Ersetzen Sie <mein_namensbereich> durch den Namensbereich für Ihre private Image-Registry. Führen Sie bx ic namespace-get aus, um Ihren Namensbereich zu finden. 
    bx ic build -t registry.domänenname/<mein_namensbereich>/nonroot.
    Anmerkung: *In diesem Befehl können Sie bx ic durch docker ersetzen, wenn Sie bei IBM Bluemix Container Service angemeldet sind und Ihre Umgebungsvariablen zur Verwendung nativer Docker-Befehle definiert haben. Sie können native Docker-Befehle in allen Schritten verwenden, die in diesem Abschnitt mit einem Stern (*) gekennzeichnet sind.
  6. Erstellen Sie einen Datenträger. 
    bx ic volume-create mein_datenträger
  7. Hängen Sie den Datenträger an und führen Sie den Container über das Image nonroot aus.* Der Datenträger-Mountpfad /mnt/myvol stimmt mit dem in der Dockerfile angegebenen Mountpfad überein. 
    bx ic run --name mein_container --volume mein_datenträger:/mnt/myvol registry.ng.bluemix.net/<mein_namensbereich>/nonroot
  8. Stellen Sie sicher, dass mein_container ausgeführt wird.* Wenn der Container den Status BUILDING (WIRD ERSTELLT) aufweist, können Sie sich nicht bei dem Container anmelden. 
    bx ic ps
    $ bx ic ps
CONTAINER ID        IMAGE                                           COMMAND             CREATED             STATUS                  PORTS                         NAMES
d92ee43d-2af        registry.domain_name/<mein_namensbereich>/nonroot:latest   ""                  41 seconds ago      Running 29 seconds ago               mein_container
  9. Melden Sie sich bei dem Container an, sobald der Container aktiv ist.* 
    bx ic exec -it mein_container /bin/bash
  10. Zeigen Sie die Berechtigungen für Ihren Datenträger-Mountpfad an. 
    ls -al /mnt/myvol/
    root@instance-006ff76b:/# ls -al /mnt/myvol/
total 12
drwxr-xr-x 3 root    root    4096 Jul 13 19:03 .
drwxr-xr-x 3 root    root    4096 Jul 13 19:03 ..
drwx------ 2 myguest myguest 4096 Jul 13 19:03 mydata

    Diese Ausgabe zeigt, dass der Rootbenutzer über Lese-, Schreib- und Ausführungsberechtigungen für den Datenträger-Mountpfad mnt/myvol/ verfügt, und der Benutzer myguest ohne Rootberechtigung über die Lese- und Schreibberechtigung für den Ordner mnt/myvol/mydata. Diese aktualisierten Berechtigungen ermöglichen dem Benutzer ohne Rootberechtigung das Schreiben von Daten auf das Persistent Volume.

Datenträgerdaten auf anderes Rechenzentrum migrieren

Wenn Sie planen, Ihren momentan verfügbaren Bluemix-Bereich auf ein neues Rechenzentrum zu verlagern, dann sollten Sie sich vergewissern, dass von Ihren Datenträgern eine Sicherungskopie erstellt wurde, sodass die Daten im neuen Rechenzentrum wiederhergestellt und verwendet werden können.

Achtung: Wenn Sie Ihren Bereich in ein neues Rechenzentrum verlagern, indem Sie den Befehl bx ic reprovision benutzen, dann werden alle Container, öffentlichen IP-Adressen und Datenträger, die sich in diesem Bereich befinden, dauerhaft entfernt und nicht auf das neue Rechenzentrum migriert. Um sicherzustellen, dass nach der Verlagerung in das neue Rechenzentrum kein Datenverlust auftritt, müssen alle Datenträger gesichert und die Daten müssen wiederhergestellt werden, nachdem Ihr Bereich im neuen Rechenzentrum wieder bereitgestellt wird.
  1. Listen Sie alle Datenträger auf, die im Bluemix-Bereich vorhanden sind und die erneut bereitgestellt werden sollen. 
    bx icvolumes
  2. Erstellen Sie für jeden Datenträger, der im neuen Rechenzentrum verwendet werden soll, eine Sicherungskopie. Wählen Sie eine der folgenden Optionen aus.
    • Verwenden Sie den Befehl bx ic cp, um den Datenträger auf Ihrem lokalen System zu sichern.
    • Erstellen Sie einen Sicherungscontainer anhand des IBM Backup and Restore-Images, um Daten in einer IBM Object Storage for Bluemix-Instanz zu sichern.
    Achtung: Wenn Sie keine Sicherungskopie Ihres Datenträgers erstellen und den Bereich für das neue Rechenzentrum erneut bereitstellen, dann gehen die Daten verloren und können nicht wiederhergestellt werden.
  3. Entfernen Sie alle Datenträger in Ihrem Bereich.
  4. Entfernen Sie alle Dateifreigaben in Ihrem Bereich.
  5. Stellen Sie Ihren Bereich im neuen Rechenzentrum erneut bereit. Ersetzen Sie dabei <rechenzentrum> durch das Rechenzentrum, in dem der neue Bereich erstellt werden soll. 
    bx ic reprovision <rechenzentrum>
  6. Erstellen Sie eine Dateifreigabe in dem neuen Bereich.
  7. Erstellen Sie einen Datenträger, den Sie zur Wiederherstellung der gesicherten Daten verwenden können.
  8. Stellen Sie die Daten auf Ihrem neuen Datenträger wieder her. Abhängig davon, wie Sie bei der Sicherung Ihres Datenträgers vorgegangen sind, können Sie zwischen den folgenden Optionen wählen.
    • Verwenden Sie den Befehl bx ic cp, um die Daten über das lokale System wiederherzustellen.
    • Erstellen Sie einen Wiederherstellungscontainer anhand des IBM Backup and Restore-Images, um Daten aus Ihrer IBM Object Storage for Bluemix-Instanz wiederherzustellen.

Datenträger über die Befehlszeilenschnittstelle (CLI) entfernen

Wenn Sie keinen Datenträger mehr benötigen, um Daten für Ihre Container zu speichern, können Sie den Datenträger entfernen.

  • Wichtig: Beim Entfernen eines Datenträgers werden alle auf dem Datenträger gespeicherten Daten oder Dateien dauerhaft entfernt. Datenträger werden nicht automatisch gesichert. Sichern Sie den Datenträger daher, bevor Sie ihn entfernen.
  1. Geben Sie den Namen des Datenträgers an. 
    bx ic volumes [--table]
  2. Untersuchen Sie den Datenträger, um zu überprüfen, an welche Container der Datenträger angehängt ist. 
    bx ic volume-inspect beispieldatenträger

    Die Ausgabe enthält einen Abschnitt mit dem Namen mounted_on_containers, in dem die Container aufgeführt sind, an die der Datenträger angehängt ist. 

        "mounted_on_containers": [
        "beispielcontainer"
    ],
  3. Wenn der Datenträger an einen Container angehängt wird, dann entfernen Sie alle vertraulichen Daten, bevor Sie den Datenträger löschen.
    1. Optional: Überprüfen Sie Ihren Container, um den Mountpfad zu ermitteln, über den der Datenträger angehängt wird. 
      bx ic inspect beispielcontainer
      Der Mountpfad befindet sich im Abschnitt Volumes der CLI-Antwort. 
      "Volumes": {
        "/mount_path": "/vol/0e65c452-62413-46f2-8b22-ff06ab74ac21/beispieldatenträger"
    2. Melden Sie sich am Container an. 
      bx ic exec -it beispielcontainer bash
    3. Navigieren Sie zu Ihrem Mountpfad. 
      cd /mountpfad
    4. Entfernen Sie alle vertraulichen Daten von Ihrem Datenträger. 
      rm -rf *
    5. Überprüfen Sie, ob alle Daten entfernt wurden. 
      ls
      Wenn keine Verzeichnisse oder Dateien angezeigt werden, dann wurden die Daten erfolgreich aus dem Datenträgermountpfad entfernt.
    6. Wiederholen Sie die Schritte a - e für alle Container, die an den Datenträger angehängt werden und deren Datenträgermountpfad vertrauliche Daten enthält.
  4. Stoppen Sie alle Container, die an Ihren Datenträger angehängt wurden, bevor Sie den Datenträger entfernen.* 
    bx ic stop beispielcontainer
    Anmerkung: *In diesem Befehl können Sie bx ic durch docker ersetzen, wenn Sie bei IBM Bluemix Container Service angemeldet sind und Ihre Umgebungsvariablen zur Verwendung nativer Docker-Befehle definiert haben. Sie können native Docker-Befehle in allen Schritten verwenden, die in diesem Abschnitt mit einem Stern (*) gekennzeichnet sind.
  5. Wenn der Container nicht mehr benötigt wird, entfernen Sie ihn.* 
    bx ic rm beispielcontainer
  6. Wenn alle Container, an die der Datenträger angehängt ist, gestoppt wurden, entfernen Sie den Datenträger. 
    bx ic volume-remove VOLNAME
    bx ic volume-remove-Parameter
    VOLNAME
    (Erforderlich) Der Name des Datenträgers.
  7. Optional: Stellen Sie sicher, dass der Datenträger entfernt wurde. 
    bx ic volumes [--table]

Dateifreigaben für Datenträger über die Bluemix-GUI hinzufügen

Eine Dateifreigabe ist ein persistentes NFS-basiertes Speichersystem (NFS - Network File System), das Docker-Datenträger in einem Bluemix-Bereich hostet und Benutzern das Speichern von sowie Zugreifen auf container- und app-bezogene Dateien ermöglicht. Um Dateien in einer Dateifreigabe speichern zu können, müssen Sie zunächst einen Containerdatenträger speichern und dann die Daten in diesem Datenträger speichern.
Anmerkung: Wenn Sie bereits die Obergrenze Ihres Kontingents erreicht haben, können Sie ein zusätzliches Kontingent anfordern, indem Sie ein Bluemix-Support-Ticket öffnen.

Da von Containern in Bluemix nicht auf den lokalen Host zugegriffen werden kann, werden die Datenträger an bereitgestellte NFS-Dateifreigaben angehängt. Die Datenträger in einer Dateifreigabe sind den Verzeichnissen auf einem Hostcomputer ähnlich. Beide Dateifreigaben und Datenträger sind pro Bereich eindeutig, so dass ein Container in demselben Bereich erstellt werden muss, um auf die persistenten Daten auf dem Datenträger zuzugreifen. Weitere Informationen zum Einrichten von Organisationen und Bereichen finden Sie unter Öffentliche Bluemix-Umgebung verwalten. Bei der ersten Erstellung eines Datenträgers in einer Organisation, wird automatisch eine Dateifreigabe von 20 GB und 4 E/A-Operationen pro Sekunde (IOPS) bereitgestellt, um den Datenträger zu hosten. Um die Leistungsanforderungen für die Daten zu erfüllen, kann der Organisationsmanager neue Dateifreigabegrößen von 20 GB bis 12 TB und mit einer Geschwindigkeit von 0,25, 2 oder 4 E/A-Operationen pro Sekunde pro GB erstellen.

  1. Wählen Sie in der Anzeige 'Organisationen verwalten' bei Ihren Kontodetails eine Organisation aus.
  2. Sehen Sie sich im Abschnitt 'Kontingent' die Details für Container an.
  3. Erstellen Sie eine Dateifreigabe. Am Ende der Tabelle wird eine Zeile erstellt.
  4. Wählen Sie einen vorhandenen Bereich aus, in dem die Dateifreigabe erstellt werden soll.
  5. Geben Sie einen Namen für die Dateifreigabe ein. Der Name darf Großbuchstaben, Kleinbuchstaben, Ziffern, Unterstreichungszeichen (_) und Bindestriche (-) enthalten.
  6. Bestimmen Sie die Kapazität Ihrer Dateifreigabeanforderungen. Die Dateifreigabegröße und die E/A-Operationen pro Sekunde pro GB (Ein-/Ausgabeoperationen pro Sekunde) werden miteinander multipliziert und bilden die Summe der E/A-Operationen pro Sekunde. Die Tabelle illustriert die Beziehung zwischen der Dateifreigabegröße und den E/A-Operationen pro Sekunde (IOPS).
    Tabelle 2. Beispiele für Dateifreigabegrößen und E/A-Operationen pro Sekunde (IOPS) pro GB
    Dateifreigabegrößen E/A-Operationen pro Sekunde pro GB Summe von E/A-Operationen pro Sekunde
    20 GB 4 80
    100 GB 4 400
    1000 GB 4 4000
    4000 GB 4 16.000
    Apps mit höheren Arbeitslasten erfordern einen höheren Wert für die Summe von E/A-Operationen pro Sekunde. Berücksichtigen Sie daher bei der Auswahl der Dateifreigabenkonfiguration, welche Apps auf Ihre Datenträger schreiben.
    1. Wählen Sie für Größe einen Wert in Gigabyte aus.
    2. Wählen Sie für E/A-Operationen pro Sekunde einen Wert pro Gigabyte aus. Basierend auf den Eingaben für Größe und E/A-Operationen pro Sekunde wird ein monatlicher Preis berechnet. Bevor Sie fortfahren, können Sie die Werte für Größe und E/A-Operationen pro Sekunde anpassen, um die monatliche Preisberechnung zu ändern.
  7. Klicken Sie auf Speichern.

Die Erstellung der Dateifreigabe kann einige Minuten dauern. Nach Abschluss der Erstellung wird das Statussymbol mit einem Häkchen angezeigt. Im angegebenen Bereich Ihrer Organisation ist nun eine Dateifreigabe verfügbar. Sie können weitere Dateifreigaben erstellen, um weitere Datenträger zu hosten, oder Sie können mehrere Datenträger auf einer Dateifreigabe hosten. Jeder Datenträger nutzt die Ressourcen der Dateifreigabe, von der er gehostet wird. Daher kann sich das Hosten mehrerer Datenträger auf einer Dateifreigabe nachteilig auf die Leistung auswirken.

Wenn Sie ein Organisationsmanager sind, können Sie als Nächstes die Abrechnung für Dateifreigaben für Ihre Organisation anzeigen.
  1. Klicken Sie auf den Benutzeravatar.
  2. Klicken Sie auf Konto.
  3. Erweitern Sie Ihre Organisation, indem Sie auf die Schaltfläche + klicken.
  4. Wählen Sie Nutzungsdashboard aus und überprüfen Sie die Nutzungs- und Abrechnungsdetails.
Weitere Informationen finden Sie unter Konto verwalten.

Dateifreigaben entfernen

Ein Organisationsmanager kann vorhandene Dateifreigaben entfernen und feststellen, welche Datenträger an eine Dateifreigabe angehängt sind.

Der Organisationsmanager kann mithilfe der Bluemix-GUI oder -CLI feststellen, welche Datenträger an eine Dateifreigabe angehängt sind, und die Dateifreigaben entfernen. Dateifreigaben können entfernt werden, wenn sie zum aktuellen Zeitpunkt nicht in der Entwicklung oder in der Produktion zum Einsatz kommen.
Anmerkung: Dateifreigaben können aufgrund des Abrechnungsverarbeitungszyklus nicht am Ende des Monats gelöscht werden. Wenn Sie am Ende des Monats beim Löschen einer Dateifreigabe eine Fehlernachricht empfangen, warten Sie bis zum ersten Tag des folgenden Monats, um die Dateifreigabe zu löschen.

Bluemix-GUI verwenden

  1. Wählen Sie in der Anzeige 'Organisationen verwalten' bei Ihren Kontodetails eine Organisation aus.
  2. Sehen Sie sich im Abschnitt 'Kontingent' die Details für Container an.
Die Dateispeichertabelle enthält eine Spalte Aktion und eine Spalte Name.
  • Klicken Sie auf den Namen der Dateifreigabe, um die angehängten Datenträger anzuzeigen.
  • Klicken Sie auf Löschen, um die Dateifreigabe zu löschen. Vor dem Löschen der Dateifreigabe müssen alle angehängten Datenträger entfernt werden. Nach dem Löschen einer Dateifreigabe wird diese dauerhaft aus Ihrem Konto entfernt. Informationen zum Entfernen eines Datenträgers finden Sie unter Datenträger über die Befehlszeilenschnittstelle (CLI) entfernen.

Befehlszeile verwenden

Führen Sie die folgenden Befehle aus, um eine Dateifreigabe über die CLI zu entfernen.
  1. Geben Sie den Namen der Dateifreigabe an.
    bx ic volume-fs
  2. Überprüfen Sie, ob Datenträger an die Dateifreigabe angehängt sind.
    bx ic volume-fs-inspect FSNAME
  3. Entfernen Sie gegebenenfalls vorhandene Datenträger.
    bx ic volume-remove VOLNAME
  4. Entfernen Sie eine vorhandene Dateifreigabe.
    bx ic volume-fs-remove FSNAME
  5. Listen Sie die vorhandenen Dateifreigaben auf, um sicherzustellen, dass die Dateifreigabe entfernt wurde.
    bx ic volume-fs
