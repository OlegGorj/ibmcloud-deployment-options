단일 및 확장 가능 컨테이너 관리를 위한 IBM Bluemix Container Service 플러그인(bx ic)

Bluemix®용 IBM® Bluemix Container Service 플러그인을 사용하여 명령행에서 이미지 또는 단일 및 확장 가능 컨테이너 관련 작업을 수행합니다.

단일 및 확장 가능 컨테이너에 기본 Docker CLI를 사용하도록 IBM Bluemix Container Service 플러그인 (bx ic) 설정

기본 Docker CLI 명령을 실행하여 컨테이너를 관리할 수 있도록 IBM Bluemix Container Service 플러그인(bx ic)을 설치합니다.
주의: 단일 및 확장 가능 컨테이너의 무료 평가판은 사용할 수 없습니다. 클러스터 관리에 대해 알아보고 Bluemix 퍼블릭에서 IBM Bluemix Container Service의 Kubernetes로 마이그레이션하십시오.

IBM Bluemix Container Service 플러그인(bx ic) 설치

명령을 실행하려면 운영 체제가 다음 요구사항을 충족해야 합니다. 이 태스크에는 이 요구사항의 설치에 대한 정보가 포함되어 있습니다. 이 도구의 이전 버전은 IBM Bluemix Container Service 플러그인(bx ic)에서 사용할 수 없습니다.

  • Docker 버전 1.8 - 1.12
  • Cloud Foundry CLI 버전 6.14.0 - 6.22.0
    참고: MacOS Sierra 사용자는 Cloud Foundry CLI 버전 6.22.0 이상을 설치해야 합니다.
  • Bluemix CLI
  • IBM Bluemix Container Service 플러그인

CLI를 설치하려면 다음을 수행하십시오.

  1. Docker 1.12를 설치하십시오. Windows 및 OS X 운영 체제의 경우 Docker는 Docker Toolbox와 함께 설치되며 여기에는 Docker Machine, Docker Engine 및 Kitematic이 포함됩니다. Linux 운영 체제의 경우 설치 메커니즘은 사용하는 Linux 배포판의 유형에 따라 다릅니다. 다음의 링크를 참조하여 Docker 1.12를 설치하십시오(Docker를 이미 설치한 경우에는 이 버전으로 업그레이드하십시오).
    참고: 컴퓨터에 이미 Docker를 설치했지만 현재 버전을 업그레이드하지 않으려는 경우에는 이 단계를 건너뛰고 Cloud Foundry CLI를 계속 설치할 수 있습니다.
    • OS X

      https://github.com/docker/toolbox/releases/download/v1.12/DockerToolbox-1.12.pkg

    • Windows

      https://github.com/docker/toolbox/releases/download/v1.12/DockerToolbox-1.12.exe

    • Linux

      사용하는 Linux 배포판에 Docker 버전 1.12를 설치하는 방법에 대한 지시사항을 찾으려면 Docker 설치 문서를 참조하십시오.

  2. Docker를 시작하십시오.
    • OS X 및 Windows의 경우 Docker Quickstart Terminal 아이콘을 두 번 클릭하십시오. Docker 디먼이 시작되고 환경 변수가 설정됩니다.
    • Linux의 경우 명령행 유틸리티에서 Docker 디먼이 실행 중인지 확인하십시오. 또한 Docker 디먼이 자동으로 다시 시작되도록 설정하십시오. 자세한 정보는 운영 체제별 Docker 문서를 참조하십시오.
  3. 터미널에서 docker run hello-word를 실행하여 Docker 설치가 올바르게 설정되었는지 확인하십시오. 
    docker run hello-world
    Docker에서 컨테이너를 실행할 hello-world 이미지를 로컬에서 검색하므로, 명령을 처음 실행하는 경우 다음 메시지가 표시됩니다. 
    Unable to find image 'hello-world:latest' locally
    이미지를 Docker Hub, 이미지의 공용 저장소에서 가져오고 명령이 완료된 후에 사용자의 출력은 다음과 같이 표시됩니다.

    명령이 적절하게 완료되지 않으면 docker version을 실행하여 Docker가 올바르게 설치되었는지 확인하십시오.
  4. GitHub 저장소에서 Cloud Foundry CLI 버전 Cloud Foundry CLI 버전 6.14.0 - 6.22.0을 설치하십시오. 운영 체제의 기본 위치에 Cloud Foundry CLI를 설치해야 합니다. 그렇지 않으면 PATH 환경 변수가 설치 디렉토리와 일치하지 않습니다.
    참고: MacOS Sierra 사용자는 Cloud Foundry CLI 버전 6.22.0 이상을 설치해야 합니다.
    Cloud Foundry CLI가 제대로 설치되었는지 확인하거나, Cloud Foundry CLI를 이미 설치한 경우 설치된 버전을 확인하려면 다음 명령을 실행하십시오.
    cf -v
  5. Bluemix CLI를 설치하십시오.
  6. Bluemix CLI에 로그인하십시오. 
    bx login -a https://api.ng.bluemix.net
  7. IBM Bluemix Container Service 플러그인을 설치하십시오. 
    bx plugin install IBM-Containers -r Bluemix
    플러그인이 올바르게 설치되었는지 확인하려면 다음 명령을 실행하십시오. 
    bx plugin list

    플러그인이 IBM-Containers로 결과에 표시됩니다.

이제 로그인하고 컨테이너 관리를 시작할 준비가 되었습니다.

IBM Bluemix Container Service CLI 플러그인(bx ic)에 로그인

CLI를 설치한 후 CLI를 사용하기 위해 로그인합니다.

시작하기 전에 다음 태스크를 완료하십시오.
  • IBM Bluemix Container Service (bx ic) 플러그인을 설치하십시오.
  • Cloud Foundry에 로그인할 때 사용할 Bluemix 조직과 영역을 결정하십시오. 조직 및 영역에 대한 자세한 정보는 조직 및 영역의 내용을 참조하십시오.
  • Cloud Foundry login 명령을 사용하여 지정할 수 있는 옵션에 대한 자세한 정보는 Cloud Foundry 문서에서 Login 명령을 참조하십시오.
  1. 명령행 유틸리티를 여십시오.
    • OS X 및 Windows의 경우 Docker Quickstart Terminal 아이콘을 두 번 클릭하십시오. Docker 디먼이 시작되고 환경 변수가 설정됩니다.
    • Linux의 경우 명령행 유틸리티를 여십시오.
  2. Cloud Foundry CLI를 통해 Bluemix에 로그인하십시오. 특정한 지역을 지정하려면 해당 지역의 API 엔드포인트와 함께 -a 옵션을 포함시키십시오. CLI를 처음으로 사용할 때 이 옵션을 포함시키지 않으면, API 엔드포인트를 입력하라는 프롬프트가 표시됩니다. API 엔드포인트가 지정되면, 이후의 로그인 시에 API 엔드포인트를 입력할 필요가 없습니다. 
    bx login [-a api.DomainName] [-sso]
    특정 지역에 로그인하려면 Cloud Foundry CLI에 로그인할 때 해당 지역에 대한 API 엔드포인트를 포함하십시오.
    • 미국 남부
      bx login -a api.ng.bluemix.net
    • 영국
      bx login -a api.eu-gb.bluemix.net
    싱글 사인온 매개변수 -sso는 연합 ID로 로그인할 때 필요합니다. 이 옵션이 사용되면 https://login.ng.bluemix.net/UAALoginServerWAR/passcode를 열고 일회성 패스코드를 얻으십시오. --sso 옵션을 포함하지 않는 경우에는 다음의 하위 단계를 완료하십시오.
    1. 이메일에 대해 Bluemix의 IBM ID를 입력하십시오.
    2. 비밀번호에 대해 IBM ID의 비밀번호를 입력하십시오. 사용자의 Bluemix 조직과 영역이 검색됩니다.
    3. Bluemix 조직 중 하나를 표시하는 번호를 입력하십시오.
    4. 기존 Bluemix 영역 중 하나를 표시하는 번호를 입력하십시오.
  3. 조직에 대한 네임스페이스를 설정하십시오. 네임스페이스는 개인용 Bluemix 이미지 레지스트리를 식별하는 고유 이름입니다. 컨테이너를 작성할 때 이미지 이름이 있는 네임스페이스를 포함하여 이미지의 위치를 지정해야 합니다.
    참고: 네임스페이스를 작성하는 경우 작성된 후에는 변경할 수 없으며 삭제만 가능합니다. Bluemix 조직의 사용자가 이미 컨테이너를 배치하고 네임스페이스를 설정한 경우 조직에 대한 네임스페이스를 표시하는 메시지가 나타납니다. 이 경우 다음 단계를 계속할 수 있습니다. 
    bx ic namespace-set <my_namespace>
    팁: 네임스페이스에 대한 다음 규칙을 준수하십시오.
    • 소문자, 숫자, 밑줄(_)만 포함할 수 있습니다.
    • 길이는 4 - 30자까지 가능합니다. 명령행에서 컨테이너를 관리할 계획이면 빨리 입력할 수 있는 짧은 네임스페이스 이름을 선호할 것입니다.
    • 해당 이름은 Bluemix에서 고유해야 합니다.
  4. IBM Bluemix Container Service를 초기화하십시오. 
    bx ic init
  5. bx ic 또는 기본 Docker 명령을 실행할 것인지 결정하십시오. 플러그인에 로그인할 때마다 이 옵션들 중 하나를 선택할 수도 있고, shellinit 명령을 사용하여 항상 옵션 2를 선택할 수도 있습니다.
    • 옵션 1(기본값)

      IBM Bluemix Container Service 랩퍼 Docker에 대한 단순한 쉘입니다. 로컬 Docker 디먼이 IBM Bluemix Container Service의 영향을 받지 않습니다. bx ic 명령이 Bluemix 영역에서 컨테이너를 관리하는 데 사용되고 docker 명령이 로컬로 컨테이너를 관리하는 데 사용됩니다. docker 명령은 로컬 이미지 개발에 항상 필요합니다.

      예를 들어, 옵션 1을 통해 bx ic run 명령으로 Bluemix 영역에 컨테이너가 작성되고 docker run으로 컨테이너가 로컬로 작성됩니다. Docker 및 IBM Bluemix Container Service 둘 다의 새 사용자이거나 로컬로 컨테이너를 테스트하려는 경우 이 옵션을 선호할 수 있습니다.

    • 옵션 2

      현재 CLI 세션의 로컬 Docker 디먼의 경로를 IBM Bluemix Container Service에 재지정하여 Bluemix 영역에서 컨테이너를 관리하는 데 사용 가능한 경우 항상 bx ic 명령의 자리에 bx ic 명령을 사용할 수 있도록 할 수 있습니다.

      예를 들어, 옵션 2를 통해 bx ic run 또는 docker run 명령을 사용자의 영역에서 컨테이너를 작성하는 데 사용할 수 있습니다. docker 명령에 익숙한 Docker 사용자인 경우 또는 IBM Bluemix Container Service에 대해 작업하는 데 자동화된 스크립트를 조정하는 경우 이 옵션을 선호할 수 있습니다.

      일부 예외에서는 bx ic 명령을 사용하여 IBM Bluemix Container Service에 특정한 태스크를 수행해야 합니다. 예를 들어, 네임스페이스를 설정하려면 bx ic namespace-set 명령을 사용해야 합니다. docker namespace-set은 올바른 명령이 아닙니다. 이 구성으로 지원되는 docker 명령의 목록은 IBM Bluemix Container Service 플러그인에 지원되는 Docker 명령(bx ic)의 내용을 참조하십시오.

      이 옵션을 구성하려면 로그인 중 터미널에서 제공되는 환경 변수를 복사하여 붙여넣으십시오.
      • DOCKER_HOST는 Docker 호스트를 IBM Bluemix Container Service로 설정합니다.
      • DOCKER_CERT_PATH는 Cloud Foundry용 IBM Bluemix Container Service 플러그인의 인증 키 위치를 설정합니다.
      • DOCKER_TLS_VERIFY는 TLS를 사용하고 원격 서버를 확인합니다.
      팁: IBM Bluemix Container Service 문서의 태스크를 완료할 때, 별표(*)로 표시된 모든 단계에서 docker 명령을 실행할 수 있습니다.

  • Pico(64MB 메모리, 4GB 디스크 공간)
  • Nano(128 MB 메모리, 8GB 디스크 공간)

(필수사항) HTTP 및 사설 네트워크 트래픽에 대한 포트를 노출합니다. 포트를 노출하지 않는 경우, 컨테이너 그룹은 실행되지만 공용 네트워크 또는 사설 네트워크 중 하나의 그룹 로드 밸런서를 통해 컨테이너 그룹에 액세스할 수 없습니다.

라우트를 바인드할 때 그룹의 컨테이너는 그룹의 노출된 포트에서 HTTP 트래픽을 청취해야 합니다. 비HTTP 포트는 공개적으로 노출할 수 없습니다. HTTPS 트래픽이 노출된 포트에 도착하면 (Go)라우터가 HTTPS 종료를 완료합니다. 그런 다음 (Go)라우터와 컨테이너 사이의 가상 사설 네트워크(VPN)에 HTTP 프로토콜이 사용됩니다. IP 주소를 바인드할 때 그룹에 있는 컨테이너가 HTTP 또는 HTTPs 트래픽을 청취하지만, 여전히 한 포트에서만 노출할 수 있습니다.

팁:

  • IBM 인증 Node.js 이미지 또는 이 이미지의 수정된 버전에 대해서는 임의의 포트(예: 8000)를 입력하십시오.
