Examen de rapport de Vulnerability Advisor sur une image

Avant de déployer une image, vous pouvez examiner son rapport Vulnerability Advisor, lequel détaille les éventuels packages vulnérables, les paramètres de conteneur ou d'application non sécurisés, et qui indique si l'image se conforme aux règles de l'organisation.

Des images de conteneurs sont fournies par IBMou peuvent être ajoutées par votre organisation. Vous devrez éventuellement configurer une image avant de la déployer afin qu'elle réponde aux exigences définies par les règles de votre organisation. Seul le responsable de l'organisation peut modifier les règles afin d'autoriser le déploiement d'une image vulnérable. Pour connaître les étapes permettant d'afficher et de modifier les règles de l'organisation, voir Examen des règles de l'organisation.

Examinez les problèmes potentiels relatifs à l'image ou à la configuration en procédant comme suit :

  1. Connectez-vous à Bluemix. Vulnerability Advisor est disponible dans l'interface utilisateur uniquement lorsque vous êtes connecté.
  2. Dans la section Evaluation des vulnérabilités, examinez le statut de votre évaluation des vulnérabilités. Le statut affiche l'une des situations suivantes :
    • Prêt à être déployé - Aucun problème de sécurité n'a été détecté.
    • A déployer avec précaution - Des problèmes de sécurité ou de configuration à corriger ont été détectés, toutefois la règle de votre organisation autorise le déploiement d'images vulnérables. Si vous déployez cette image, vous recevez un avertissement.
    • Déploiement bloqué - Des problèmes de sécurité ou de configuration ont été détectés et doivent être corrigés avant que l'image ne puisse être déployée vu que votre règle d'organisation n'autorise pas le déploiement d'images vulnérables.
    • Incomplet - L'analyse est incomplète. Il se peut qu'elle soit encore en cours ou que le système d'exploitation de l'image soit incompatible. Patientez et relancez l'analyse. Si l'analyse est toujours incomplète, transférez à nouveau l'image pour lancer une nouvelle analyse. Le déploiement des images dont l'analyse est incomplète n'est pas bloqué.

    Passez en revue le rapport détaillé qui constitue l'évaluation de sécurité de Vulnerability Advisor :

  3. Sélectionnez l'image et dans la section Evaluation des vulnérabilités, cliquez sur Afficher le rapport.
  4. Passez en revue les différentes sections pour identifier les problèmes de sécurité et de configuration potentiels pour chaque package de l'image :
    • Règles de l'organisation

      Indique si l'image est conforme aux règles définies par le responsable de l'organisation.

    • Analyse des risques

      Si l'image comporte au moins une vulnérabilité, cette section affiche l'évaluation des risques telle qu'elle est calculée par l'assistant de détection des vulnérabilités. Le risque est calculé à l'aide de la calculatrice CVSS et la technologie IBM® X-Force Exchange.

    • Packages vulnérables

      Recense les packages affectés par des problèmes de vulnérabilité connus, lesquels sont mis à jour quotidiennement depuis les avis de sécurité publiés pour les types d'image Docker répertoriés dans Gestion de la sécurité des conteneurs et des images avec Vulnerability Advisor. Généralement, pour qu'un package vulnérable réussisse l'examen, une version plus récente de ce package incluant un correctif pour la vulnérabilité est requise. Un même package peut répertorier plusieurs vulnérabilités, auquel cas, une même mise à jour peut corriger plusieurs problèmes. Cliquez sur le code de l'avis de sécurité pour consulter plus d'informations sur le package et sur la procédure à suivre pour le mettre à jour.

    • Paramètres de conteneur

      Répertorie les suggestions qui peuvent vous permettre d'augmenter la sécurité de l'image. La colonne Corrective Action explique comment améliorer la sécurité.

    • Configurations d'applications

      Recense les paramètres d'application non sécurisés de l'image. La colonne Corrective Action explique comment améliorer la sécurité.

    • Instances de conteneur

      Recense les conteneurs utilisant cette image et déjà en opération dans votre organisation.

    Des actions correctives ou des suggestions sont indiquées pour chaque problème répertorié.

  5. Résolvez les problèmes décrits dans le rapport Evaluation des vulnérabilités et régénérez l'image. Certains problèmes consignés dans le document Dockerfile peuvent être résolus à l'aide du code fourni dans Résolution des vulnérabilités dans les conteneurs ou les images.
Après le déploiement de votre image, Vulnerability Advisor continue à être à l'affût de problèmes de sécurité et de configuration dans le conteneur. Vous pouvez résoudre les problèmes rencontrés en suivant la procédure décrite dans la section Examen des vulnérabilités des conteneurs.
