Examen des vulnérabilités des conteneurs

Dans votre tableau de bord, vous pouvez examiner le statut d'un conteneur pour déterminer si sa sécurité se conforme aux règles de votre organisation. Vous pouvez également consulter un rapport des vulnérabilités d'un conteneur, lequel recense les éventuels packages vulnérables et les paramètres de conteneur ou d'application non sécurisés.

Vérifiez que les conteneurs s'exécutant dans votre espace continuent à se conformer aux règles de votre organisation en examinant la zone Statut de la règle. Le statut affiche l'une des situations suivantes :
  • Réussite - Aucune vulnérabilité n'a été détectée.
  • Avertissement - Vulnerability Advisor a détecté des vulnérabilités potentielles dans votre conteneur qui entraînent que votre conteneur n'est pas en conformité avec la règle. Si la règle de votre organisation permet le déploiement d'images vulnérables, il se peut que l'image ait été déployée avec l'état (Déployer avec précaution) et qu'un avertissement ait été envoyé à l'utilisateur l'ayant déployé.
  • Violation de règle - Vulnerability Advisor a détecté des vulnérabilités potentielles dans votre conteneur qui entraînent que votre conteneur n'est pas en conformité avec la règle. La règle de votre organisation n'autorise pas le déploiement d'images vulnérables.
  • Incomplet - L'analyse est incomplète. Il se peut que l'analyse soit encore en cours ou que le système d'exploitation pour cette instance de conteneur ne soit pas compatible.

Si Statut de la règle n'indique pas Réussite, vérifiez que votre conteneur est sécurisé autant que possible en examinant son rapport de vulnérabilité et en corrigeant les vulnérabilités signalées en procédant comme suit :

  1. Dans votre tableau de bord Bluemix, sélectionnez le conteneur que vous désirez examiner.
  2. Dans le panneau Vulnerability Advisor, cliquez sur Afficher le rapport.
  3. Passez en revue les sections pour voir les vulnérabilités potentielles de chaque package contenu dans l'image :
    • Règles de l'organisation

      Indique si le conteneur adhère à la règle définie par le responsable de l'organisation.

    • Packages vulnérables

      Répertorie les packages avec des problèmes de vulnérabilité connus (mis à jour quotidiennement à partir des avis de sécurité publiées pour les systèmes d'exploitation répertoriés dans Recherche de vulnérabilités d'image à l'aide de Vulnerability Advisor). Généralement, pour qu'un package vulnérable réussisse l'examen, une version plus récente de ce package incluant un correctif pour la vulnérabilité est requise. Un même package peut répertorier plusieurs vulnérabilités, auquel cas, une même mise à jour peut corriger plusieurs problèmes. Cliquez sur le code de l'avis de sécurité pour consulter plus d'informations sur le package et sur la procédure à suivre pour le mettre à jour.

    • Paramètres de conteneur

      Recense des suggestions que vous pouvez suivre pour améliorer la sécurité du conteneur. La colonne Corrective Action explique comment améliorer la sécurité.

    • Configurations d'applications

      Répertorie les paramètres d'application non sécurisés pour le conteneur. La colonne Corrective Action explique comment améliorer la sécurité.

    • Source de l'image Docker

      Affiche l'image qui a été utilisée pour créer ce conteneur.

    Des actions correctives ou des suggestions sont indiquées pour chaque problème répertorié.

  4. Décidez comment mettre à jour le conteneur en vue de résoudre les problèmes. Vous devez avoir une bonne connaissance de l'architecture de votre conteneur pour pouvoir choisir la méthode appropriée pour le mettre à jour. Exemple :
    • Si votre conteneur est découplé des données qu'il traite, vous pouvez arrêter le conteneur, apporter les modifications requises à l'image, puis le redéployer.
    • Vous pouvez utiliser Active Deploy pour mettre à jour l'instance de conteneur vulnérable sans arrêter le conteneur.
    • Dans une configuration à haute disponibilité avec plusieurs conteneurs dans un groupe de conteneur, vous pourriez acheminer le trafic à une autre instance de conteneur pendant que vous corrigez les vulnérabilités.
  5. Corrigez les problèmes décrits dans le rapport Evaluation des vulnérabilités et régénérez l'image ou redéployez le conteneur, selon la méthode retenue. Certains problèmes consignés dans le document Dockerfile peuvent être résolus à l'aide du code fourni dans Résolution des vulnérabilités dans les conteneurs ou les images.
