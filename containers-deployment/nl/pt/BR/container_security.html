Segurança para contêineres únicos e escaláveis

O IBM® Bluemix Container Service fornece uma plataforma de nuvem confiável e segura que otimiza recursos de segurança do Linux e do Docker. Ao construir e executar contêineres com o IBM Bluemix Container Service, seu app se beneficia do isolamento de contêiner e rede, acesso limitado a dados e recursos do host e implementações seguras do contêiner.
Atenção: Avaliações grátis para o contêineres únicos e escaláveis não estão disponíveis. Saiba mais sobre gerenciamento de cluster e migração para o Kubernetes no IBM Bluemix Container Service no Bluemix Public.

Configurações de segurança no host de cálculo do IBM Bluemix Container Service

O IBM Bluemix Container Service otimiza o kernel do Linux e recursos de segurança do Docker para assegurar o isolamento do contêiner e para restringir o acesso aos recursos de host de cálculo.

Daemons do Docker totalmente gerenciados e seguros
Os daemons do Docker que são executados nos hosts de cálculo da IBM são configurados sem acesso de usuário direto e podem ser configurados somente pela IBM. Todos os soquetes de daemon do Docker são protegidos com certificados TLS.
Namespaces ativados do usuário do Docker
O recurso de namespace do usuário do Docker está ativado no Docker Engine nos hosts de cálculo. O recurso de namespace do usuário isola os processos que são executados dentro de um contêiner de todos os outros processos e contêineres que são executados no mesmo host de cálculo. O usuário raiz efetiva dentro do contêiner é mapeado para um usuário não raiz no host de cálculo. Mesmo que um usuário raiz gerenciado para divisão do contêiner, o usuário não tem qualquer permissão para manipular arquivos, processos ou outros contêineres executados no host de cálculo.
Contêineres privilegiados não permitidos
A opção --privileged no comando run não é suportada. Os contêineres não têm acesso a dispositivos e discos rígidos no host de cálculo.
Grupos de controle gerenciados (cgroups) para memória, CPU e espaço em disco
Os contêineres que são executados no mesmo host de cálculo compartilham memória, CPU e espaço em disco. Para evitar que um contêiner consuma todos os recursos disponíveis ou que reduza aplicativos ou sistemas host, a quantidade máxima de memória, CPU e espaço em disco que pode ser designada a um contêiner é limitada.

A quantidade de memória, CPU e espaço em disco que está disponível ao contêiner durante o tempo de execução é determinada pelo tamanho do contêiner selecionado durante a criação do contêiner. Essa limitação assegura que cada contêiner no host de cálculo obtenha seu compartilhamento justo dos recursos disponíveis do host. Se um contêiner não se comportar bem, a limitação de recursos assegurará um desempenho e comportamento consistentes para outros contêineres no host de cálculo.

Otimização de kernel do host de cálculo
Os kernels do host de cálculo limitam o número total de encadeamentos e processos que podem ser executados no host de cálculo por um usuário ou grupo específico. Essa otimização assegura que o host não seja sobrecarregado e não fique sem intervalos de processo até que o máximo do usuário seja atingido.
Monitoramento contínuo de host de cálculo
Todo host de cálculo é monitorado continuamente pela IBM para controlar e corrigir fork-bombs e outros ataques de Negação de Serviço (DOS) em nível de processo.
Perfil AppArmor ativado para contêineres do Docker
Cada host de cálculo aplica o perfil AppArmor específico da IBM para proteger o ambiente e restringir os recursos de um contêiner no host de cálculo. O AppArmor é um módulo de segurança do kernel do Linux que controla o acesso a pastas, arquivos, domínios de rede e permissões para criar e mudar dados.
Montagem de volume local desativada em hosts de cálculo
O IBM Bluemix Container Service não suporta a montagem de volumes locais ou de diretórios locais/do host para um contêiner. Em vez disso, os volumes com escopo na organização são usados para persistir dados entre reinicializações do contêiner e para tornar os dados disponíveis para múltiplos contêineres. Os volumes são hospedados em compartilhamentos de arquivo isolados que armazenam com segurança dados de app e gerenciam o acesso e a permissão aos arquivos.

Os volumes com escopo definido na organização devem ser criados para poder montá-los no contêiner. É possível criar um volume na GUI do Bluemix ou na CLI.

Configurações de segurança na rede de contêiner privada do IBM Bluemix Container Service

Revise a lista de recursos integrados do IBM Bluemix Container Service que asseguram comunicação segura entre os contêineres na mesma rede privada, entre contêineres em diferentes espaços do Bluemix e entre seu espaço do Bluemix e uma rede remota.

Rede de contêiner isolada L2 privada por espaço
Uma rede privada do contêiner cria um ambiente isolado e seguro para os contêineres únicos e os grupos de contêineres que são executados em um espaço. Os contêineres que são conectados à mesma rede privada podem enviar e receber dados de outros contêineres na rede privada usando os endereços IP privados. Os contêineres não ficam disponíveis publicamente até que uma porta pública e um endereço IP público para contêineres únicos ou uma rota pública para grupos de contêineres sejam ligados.

No Bluemix, cada espaço é fornecido com uma rede privada do contêiner em que as configurações de rede padrão do IBM Bluemix Container Service são aplicadas. Ao criar contêineres, eles são conectados automaticamente à rede privada padrão para o espaço e um endereço IP privado é designado a eles.

Manter os contêineres privados ou expor seletivamente serviços voltados ao público por meio de Grupos de segurança de rede
Cada contêiner é protegido por Grupos de segurança de rede específicos da IBM que controlam o tráfego de rede de entrada e servem como firewall de software no nível de gateway de rede privada. Os dados que são enviados por meio do gateway de rede privada são verificados quanto à conformidade com relação às regras do Grupo de segurança de rede e podem ser passados para o contêiner destinatário ou bloqueados.

Ao criar um contêiner e não expô-lo publicamente, ele é configurado com o Grupo de segurança de rede privada padrão. Esse grupo de segurança permite a comunicação de rede privada entre os contêineres usando os endereços IP privados. Por padrão, todas as portas do contêiner são expostas automaticamente na rede privada. Não é necessário mapear as portas do contêiner para uma porta do host.

Para tornar seu aplicativo disponível publicamente, deve-se expor uma porta pública e ligar um endereço IP público ao contêiner único ou uma rota pública ao seu grupo de contêiner. Ao expor uma porta pública usando a opção -p no comando bx ic run, você cria um Grupo de segurança de rede pública para seu contêiner que permite o envio e o recebimento de dados públicos na porta exposta somente. Todas as outras portas públicas são fechadas e não podem ser usadas para acessar seu app na Internet.

Monitoramento de rede corporativa contínuo
Todo o tráfego de rede no IBM Bluemix Container Service é monitorado pela IBM para detectar e corrigir atividades maliciosas.
Suporte completo para os comandos exec, attach e logs do Docker
Você pode efetuar login no contêiner ou visualizar a saída detalhada durante o tempo de execução utilizando os comandos bx ic exec, bx ic attach ou bx ic logs. Com esses comandos, você pode evitar a execução de um daemon SSH não seguro para acessar e monitorar o contêiner durante o tempo de execução.

Para obter mais informações sobre como efetuar login com segurança em seu contêiner usando bx ic exec, consulte Efetuando login em um contêiner com exec.

VPN (rede privada virtual) para acesso de rede privada segura a serviços em outros espaços e em qualquer rede remota
Os contêineres em uma rede privada de contêiner podem ser conectados com segurança a datacenters corporativos usando o serviço IBM® Virtual Private Network.

Para obter mais informações, veja Configurando uma conexão VPN entre seus contêineres e um datacenter corporativo

Efetuando login em um contêiner com exec

Se você tiver que efetuar login no contêiner em execução, será possível usar bx ic exec.

Os usos comuns do comando bx ic exec incluem abrir uma janela bash e visualizar arquivos de um volume designado a um contêiner. O comando bx ic exec somente é executado em um contêiner que está em um estado de execução. Para manter um contêiner funcionando, deve-se iniciá-lo com um comando de longa execução. Se um contêiner for pausado ou interrompido, ele deverá ser reiniciado com um comando de longa execução antes que se possa efetuar login nele.

Comando e parâmetros

bx ic exec [-d] [-it ] [-u USER] CONTAINER CMD
Nota: *Nesse comando, será possível substituir bx ic por docker ao efetuar login no IBM Bluemix Container Service e configurar suas variáveis de ambiente para usar comandos nativos do Docker.

Exemplos

Este exemplo de comando executa um terminal bash interativo no contêiner my_container. 
bx ic exec -it my_container bash
Este exemplo de comando executa o comando de data no contêiner my_container. 
bx ic exec my_container date

Conectando contêineres a um datacenter corporativo configurando uma Rede privada virtual

Conecte com segurança os contêineres únicos e os grupos de contêineres em uma rede de contêiner privada no Bluemix a um datacenter corporativo usando o serviço IBM® Virtual Private Network (VPN). O IBM VPN fornece um canal de comunicação seguro de ponta a ponta na Internet que se baseia no conjunto de protocolo Internet Protocol Security (IPsec) padrão de mercado. O protocolo IPsec oferece autenticação de peer no nível de rede, integridade de dados e confidencialidade de dados, criptografando os pacotes que são trocados entre os terminais VPN. Para configurar uma conexão segura entre os contêineres no Bluemix e um datacenter corporativo, deve-se ter um gateway VPN IPsec ou um servidor SoftLayer instalado no datacenter local. Com o serviço IBM VPN, é possível configurar um gateway VPN por espaço e definir até 16 conexões com destinos diferentes.

Para conectar contêineres a um datacenter corporativo, siga estas etapas.

  1. Crie pelo menos um contêiner único ou um grupo de contêiner em seu espaço que você queira conectar ao datacenter corporativo.
  2. Crie uma instância do serviço IBM VPN em seu espaço seguindo a etapa 1-5 do tópico Ligando um serviço.
    Nota: A instância de serviço IBM VPN deve ser criada no mesmo espaço em que você criou os contêineres que deseja conectar ao datacenter corporativo.
  3. Siga as instruções no tópico Introdução do IBM VPN para configurar o gateway VPN e configurar as conexões com o datacenter corporativo.
